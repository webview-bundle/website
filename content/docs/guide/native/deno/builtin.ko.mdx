---
title: '내장 번들'
description: 번들을 BundleSource가 읽는 빌트인 디렉터리에 설치하여 Deno Desktop 앱 안에 함께 배포합니다.
---

빌트인 번들은 앱 안에 함께 배포되므로, 아직 원격으로 다운로드한 번들이 없는 첫 실행 시점에도 창이 오프라인으로 로드됩니다. `BundleSource`는 `builtinDir`에서 이 번들들을 읽고, `wvb builtin`은 설치되는 번들 세트를 그 디렉터리에 기록합니다.

## 소스를 빌트인 디렉터리로 지정하기

`webviewBundle`은 `source`로부터 `BundleSource`를 만듭니다. `builtinDir`에는 번들을 설치할 디렉터리를 지정하세요. 기본값은 엔트리 모듈 옆의 `bundles`입니다.

```ts title="main.ts"
import { webviewBundle, bundleProtocol } from '@wvb/deno-desktop';

const wvb = await webviewBundle({
  protocols: [bundleProtocol('app')],
  source: { builtinDir: './bundles' },
});
```

## wvb builtin으로 설치하기

`wvb builtin`을 실행하고 `--out`을 같은 디렉터리로 지정하세요. 이 명령은 `manifest.json` 하나와, 번들마다 `<name>/<name>_<version>.wvb` 파일 하나씩을 기록합니다.

```sh
wvb builtin --out ./bundles
```

번들을 리모트 엔드포인트에서 가져올지 로컬 워크스페이스에서 가져올지는 CLI 플래그가 아니라 [설정](/docs/references/configuration)의 `builtin.target`으로 정합니다. 기본값은 리모트 타깃입니다. 모든 플래그와 두 가지 타깃 형태는 [`wvb builtin`](/docs/references/cli/builtin)을 참고하세요.
