<Badge>실험적</Badge>

개발 중에는 `bundleProtocol`을 `localProtocol`로 바꾸세요. 웹뷰를 개발 서버에 프록시하므로, 패킹된 `.wvb` 자산을 로드하는 대신 수정 사항이 라이브로 리로드됩니다.

## 개발 서버로 프록시하기

`localProtocol(scheme, { hosts })`은 제공되는 host를 개발 서버의 base URL로 매핑합니다. 이를 Vite(또는 프런트엔드를 제공하는 무엇이든)로 향하게 하고, 프로덕션과 동일한 단일 origin `webviewBundle` 연결을 그대로 재사용하세요.

```ts title="main.ts"
import { webviewBundle, localProtocol } from '@wvb/deno-desktop';

const wvb = await webviewBundle({
  protocols: [
    localProtocol('app', {
      hosts: { app: 'http://localhost:5173' },
    }),
  ],
});

const server = Deno.serve(wvb.fetch);
const win = new Deno.BrowserWindow({ url: `http://localhost:${server.addr.port}` });
await win.closed;
```

## 프로덕션으로 되돌리기

패킹된 `.wvb` 자산을 오프라인으로 제공하려면 `localProtocol`을 다시 `bundleProtocol`로 바꾸세요. 프로덕션 연결은 [설정](/docs/guide/native/deno)을 참고하세요.
