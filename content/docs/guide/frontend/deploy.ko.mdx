업로드는 버전을 리모트에 저장합니다. 배포는 그 버전을 **현재** 버전, 즉 클라이언트가 원격으로 다운로드하는 버전으로 만듭니다. 배포는 별도의 빠른 단계로, 다시 패킹하거나 다시 업로드하지 않습니다.

[업로드](/docs/guide/frontend/upload) 뒤에 배포하거나, `wvb upload`에 `--deploy`를 전달해 둘을 한 번에 처리할 수 있습니다.

## 버전 배포하기

이미 업로드된 버전으로 리모트를 지정합니다.

```sh
wvb deploy app --version 1.2.0
```

`app`은 번들 이름입니다(설정이나 `package.json` 값으로 대체됩니다). 버전은 `--version`(`-V`) 플래그로 지정하며, 위치 인자로 버전을 넣는 방식은 없습니다.

## 채널

이름이 지정된 채널로 배포하면 릴리스 트랙(`stable`, `beta`, `internal`, …)을 따로 유지할 수 있습니다. 해당 채널의 클라이언트는 그 채널에 배포한 버전을 받습니다.

```sh
wvb deploy app --version 1.2.0 --channel beta
```

`--channel`을 생략하면 기본 채널로 배포됩니다.

## 롤백

전용 롤백 명령어는 없습니다. 롤백하려면 이전에 업로드해 둔 버전을 배포하세요. 그 버전이 다시 현재 버전이 됩니다.

```sh
wvb deploy app --version 1.1.0
```

리모트에 이미 올라와 있는 버전은 무엇이든 다시 배포할 수 있으므로, 문제없이 동작하는 버전들을 업로드된 상태로 유지하세요.
