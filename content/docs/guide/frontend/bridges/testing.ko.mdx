네이티브 호스트 없이 브릿지를 호출하는 웹 코드를 테스트합니다. `@wvb/bridge/testing` 서브패스는 브릿지 호출을 스텁 처리하여, 실제 플랫폼 대신 픽스처를 기준으로 코드가 결과를 반환하도록 합니다.

## 커맨드 스텁 처리하기

`mockInvoke(command, handler)`은 점으로 구분된 커맨드 키 하나에 핸들러를 등록하고, 해제 가능한 핸들을 반환합니다. 핸들러는 메서드의 params를 위치 인자로 받고, 호출이 반환할 값을 돌려줍니다.

```ts title="load-version.test.ts"
import { expect, test } from 'vitest';
import { source } from '@wvb/bridge';
import { mockInvoke } from '@wvb/bridge/testing';

test('loads the current version', async () => {
  using _mock = mockInvoke('source.loadVersion', bundleName => ({
    type: 'remote',
    version: '1.0.0',
  }));

  await expect(source.loadVersion('app')).resolves.toEqual({
    type: 'remote',
    version: '1.0.0',
  });
});
```

<Callout type="info">
  `using`은 블록이 끝날 때 목을 해제합니다. `using` 없이 사용한다면, 스텁이 다음 테스트로 새어
  나가지 않도록 `_mock[Symbol.dispose]()`나 `_mock.clear()`을 직접 호출하세요.
</Callout>
