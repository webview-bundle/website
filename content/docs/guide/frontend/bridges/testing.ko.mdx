네이티브 호스트 없이 브릿지를 호출하는 웹 코드를 테스트합니다. `@wvb/bridge/testing` 서브패스는 브릿지 호출을 목 처리하여, 실제 플랫폼 대신 픽스처를 기준으로 코드가 결과를 반환하도록 합니다.

## 브릿지 목 만들기

`mockBridge(options?)`은 핸들러를 등록하는 해제 가능한 브릿지를 생성합니다. 점으로 구분된 커맨드 키마다 `.mockInvoke(command, handler)`을 체이닝하세요. 핸들러는 메서드의 params를 위치 인자로 받고, 호출이 반환할 값(또는 그 값의 프로미스)을 돌려줍니다.

```ts title="load-version.test.ts"
import { expect, test } from 'vitest';
import { source } from '@wvb/bridge';
import { mockBridge } from '@wvb/bridge/testing';

test('loads the current version', async () => {
  using bridge = mockBridge({ platform: 'android' });
  bridge.mockInvoke('source.loadVersion', bundleName => ({
    type: 'remote',
    version: '1.0.0',
  }));

  await expect(source.loadVersion('app')).resolves.toEqual({
    type: 'remote',
    version: '1.0.0',
  });
});
```

플랫폼을 함께 목 처리하려면 `{ platform }`을 전달하세요.

```ts
using bridge = mockBridge({ platform: 'ios' }).mockInvoke('source.loadVersion', () => ({
  type: 'remote',
  version: '1.0.0',
}));
```

<Callout type="info">
  `using`은 블록이 끝날 때 브릿지를 해제하며, 등록한 모든 목을 정리합니다. `using` 없이 사용한다면
  스텁이 다음 테스트에 영향가지 않도록 `bridge.clear()`을 직접 호출하세요.
</Callout>

## 플랫폼 목 처리하기

`mockPlatform(type)`은 `platform.type`(`platform.isIos`, `platform.isElectron` 등)으로 분기하는 코드를 위해 플랫폼 감지를 단독으로 재정의합니다. 스코프가 끝나면 이전 플랫폼을 복원하는 해제 가능한 핸들을 반환합니다. `type`은 `'electron'`, `'tauri'`, `'android'`, `'ios'`, `'deno'` 중 하나입니다.

```ts
import { platform } from '@wvb/bridge';
import { mockPlatform } from '@wvb/bridge/testing';

test('detects the iOS platform', () => {
  using _platform = mockPlatform('ios');
  expect(platform.isIos).toBe(true);
});
```

## 단일 커맨드 스텁 처리하기

`mockInvoke(command, handler)`은 브릿지 인스턴스 없이 커맨드 하나를 등록하며, `mockBridge`이 기반으로 삼는 기본 API입니다. 해제 가능한 핸들을 반환하며, 등록된 목을 모두 초기화하려면 `afterEach` 등에서 `clearInvokeMocks()`을 호출하세요.

```ts
using _mock = mockInvoke('source.loadVersion', () => ({ type: 'remote', version: '1.0.0' }));
```
