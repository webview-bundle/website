`@wvb/bridge`은 웹 측 패키지입니다. 웹뷰 안의 JavaScript는 이 패키지를 임포트해서 번들 소스와 리모트, 업데이터를 관리하는 네이티브 호스트를 호출합니다. 플랫폼마다 트랜스포트를 자동으로 감지하므로, 동일한 웹 코드가 Electron, Tauri, Android, iOS에서 그대로 실행됩니다.

```ts
import { source, remote, updater } from '@wvb/bridge';
```

## 세 가지 표면

- `source` — 네이티브 번들 소스를 확인하고 관리합니다: 번들 목록 조회, 버전 로드 및 전환, 파일 해석.
- `remote` — 호스트를 통해 설정된 리모트와 통신합니다: 번들 목록 조회, 확인, 다운로드.
- `updater` — 원격 업데이트 흐름을 구동합니다: 업데이트 확인, 다운로드, 설치.

각 표면은 타입이 지정되어 있으며 저수준 `invoke()` 위에 구축됩니다. 전체 API는 [브릿지 레퍼런스](/docs/references/javascript/bridge)를 참고하세요.

## 언제 사용하나요

웹 코드가 어떤 번들이 제공되는지 알아야 하거나 바꿔야 할 때 `@wvb/bridge`을 사용하세요. 활성 소스를 확인하거나, 리모트에서 더 새로운 번들을 가져오거나, 앱 안에서 원격 업데이트를 실행하는 경우입니다.
