---

배포된 번들의 현재 버전을 다운로드합니다. 서버는 `.wvb` 아카이브를 응답 본문으로 스트리밍하고, 번들의 메타데이터를 `Webview-Bundle-*` 헤더에 담아 전달합니다. [업데이터](/docs/references/api/node/updater)는 검사 결과 새 버전이 있다고 확인되면 이 호출로 새 버전을 가져옵니다.

```http
GET /bundles/{name}
```

기본 배포 대신 특정 채널의 현재 버전을 다운로드하려면 `?channel=`을 전달하세요.

## 매개변수

| Name      | In    | Type   | Required | Description                                       |
| --------- | ----- | ------ | -------- | ------------------------------------------------- |
| `name`    | path  | string | yes      | 다운로드할 번들 이름.                             |
| `channel` | query | string | no       | 선택할 채널. 생략하면 기본 배포를 다운로드합니다. |

## 응답

| Status | Description                                      |
| ------ | ------------------------------------------------ |
| `200`  | 번들을 찾았습니다. 본문은 `.wvb` 아카이브입니다. |
| `404`  | 해당 이름의 번들이 배포되지 않았습니다.          |

`404` 형태와 클라이언트가 그 밖의 non-2xx 응답을 매핑하는 방식은 [오류](/docs/guide/remote/http-spec/errors)를 참고하세요.

### 응답 헤더

`200` 응답은 [HEAD /bundles/{name}](/docs/guide/remote/http-spec/endpoints/head-current)과 동일한 번들 메타데이터 헤더를 담습니다. 클라이언트는 헤더를 대소문자 구분 없이 읽습니다. 여기서는 표준 표기를 보여줍니다.

| Header                     | Required | Description                       |
| -------------------------- | -------- | --------------------------------- |
| `Webview-Bundle-Name`      | yes      | 번들 이름.                        |
| `Webview-Bundle-Version`   | yes      | 번들 버전.                        |
| `Webview-Bundle-Integrity` | no       | 무결성 문자열 `"<alg>:<base64>"`. |
| `Webview-Bundle-Signature` | no       | Base64 서명.                      |
| `ETag`                     | no       | 표준 검증자.                      |
| `Last-Modified`            | no       | 표준 검증자.                      |

클라이언트는 `Webview-Bundle-Name` 또는 `Webview-Bundle-Version`이 빠진 `200` 응답을 거부합니다.

응답 본문은 원본 `.wvb` 아카이브 바이트이며, `Content-Type: application/webview-bundle`로 제공됩니다.

## 예제

현재 버전을 다운로드하고 응답 헤더를 출력합니다.

```sh
curl -s http://localhost:4313/bundles/app -o app.wvb -D -
```

```http
HTTP/1.1 200 OK
Content-Type: application/webview-bundle
Webview-Bundle-Name: app
Webview-Bundle-Version: 1.2.0
Webview-Bundle-Integrity: sha256:n4bQgYhMfWWaL+qgxVrQFaO/TxsrC4Is0V1sFbDwCgg=

<binary .wvb bytes>
```

`?channel=`로 채널의 현재 버전을 선택합니다.

```sh
curl -s 'http://localhost:4313/bundles/app?channel=beta' -o app-beta.wvb
```

배포되지 않은 번들은 `404`를 반환합니다.

```sh
curl -sI http://localhost:4313/bundles/unknown
```

```http
HTTP/1.1 404 Not Found
```
