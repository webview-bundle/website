AWS 프로바이더는 CloudFront 배포 뒤의 S3 버킷에서 번들을 제공합니다. 두 개의 Lambda@Edge 함수가 엣지에서 리모트 HTTP 규약을 구현합니다.

## 서버 프로비저닝

`@wvb/remote-aws-provider-pulumi`은 버킷, 배포, 엣지 함수까지 전체 스택을 [Pulumi](https://www.pulumi.com) 컴포넌트로 프로비저닝합니다.

```sh
npm install @wvb/remote-aws-provider-pulumi
```

```ts title="index.ts"
import { WebviewBundleRemoteProvider } from '@wvb/remote-aws-provider-pulumi';

const remote = new WebviewBundleRemoteProvider('wvb', {
  bucketName: 'my-bundles',
  // allowOtherVersions: true,  // GET /bundles/{name}/{version}도 제공
});

export const endpoint = remote.cloudfrontDistributionDomainName;
```

요청 핸들러 자체는 `@wvb/remote-aws-provider`(`webviewBundleRemote`)이며, 컴포넌트가 대신 배포합니다. 엣지 함수를 직접 연결하려면 [패키지 레퍼런스](/docs/references/javascript/remotes/remote-aws-provider)를 참고하세요.

## 게시하기

`@wvb/remote-aws`은 `uploader`과 `deployer`, 그리고 선택적인 AWS KMS `signature` 서명기를 설정용으로 제공합니다.

```sh
npm install -D @wvb/remote-aws
```

```ts title="wvb.config.ts"
import { defineConfig } from '@wvb/config';
import { awsRemote } from '@wvb/remote-aws';

export default defineConfig({
  remote: {
    endpoint: 'https://d111111abcdef8.cloudfront.net',
    integrity: { algorithm: 'sha256' },
    ...awsRemote({
      bucket: 'my-bundles',
      // deployer: { invalidation: { distributionId: 'E1234567890ABC' } },
      // signature: { keyId: 'arn:aws:kms:…', algorithm: 'ECDSA_SHA_256' },
    }),
  },
});
```

`awsRemote()`은 `{ uploader, deployer, signature? }`을 반환합니다. 자격 증명과 리전은 표준 AWS SDK 체인을 따르며, `aws` 옵션으로 재정의할 수 있습니다. 모든 옵션은 [AWS 클라이언트 레퍼런스](/docs/references/javascript/remotes/remote-aws)를 참고하세요.

<Callout type="info">
  서버 옵션은 `bucketName`, 클라이언트 옵션은 `bucket`입니다. 둘 다 같은 버킷을 가리켜야 합니다.
  Lambda@Edge는 버킷 리전과 무관하게 항상 `us-east-1`에서 실행됩니다.
</Callout>
