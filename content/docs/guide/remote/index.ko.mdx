리모트는 앱이 새 번들을 원격(Over-The-Air)으로 다운로드하는 HTTP 서버입니다. 패킹한 `.wvb`을 리모트에 업로드하고 현재 버전을 지정하면, 클라이언트는 네이티브 앱을 새로 배포하지 않고도 해당 버전을 받아옵니다.

## 게시와 다운로드

게시는 두 단계로 이뤄지며, 모두 [CLI](/docs/guide/remote/using-cli)에서 실행합니다.

1. **업로드**은 패킹한 버전을 리모트에 저장합니다.
2. **배포**은 클라이언트가 받을 현재 버전을 지정합니다.

이후 클라이언트는 업데이터를 통해 배포된 버전을 확인하고 다운로드합니다. 클라이언트 쪽 흐름은 [Over-The-Air](/docs/guide/core-concepts/over-the-air) 문서를, 이를 연결하는 `remote` 블록은 [리모트 설정](/docs/references/configuration/remote)을 참고하세요.

## 리모트 실행

리모트를 실행하는 방법은 두 가지입니다.

**프로바이더 사용하기.** 프로바이더는 배포할 서버와, 설정에 넣을 `uploader`/`deployer` 쌍을 제공합니다. HTTP 규약을 직접 구현할 필요가 없습니다.
