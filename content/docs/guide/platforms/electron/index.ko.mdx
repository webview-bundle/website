`@wvb/electron`은 세 단계로 Webview Bundle을 Electron에 연결합니다. 커스텀 프로토콜을 통해 UI를 `.wvb` 번들에서 제공하고, 개발 중에는 개발 서버로 프록시하며, 필요하면 번들을 무선(OTA)으로 업데이트합니다.

## Install

`@wvb/electron`은 Electron 15 이상을 요구하며 `@wvb/node`(사전 빌드된 N-API 바이너리 — Rust 툴체인 불필요)를 함께 설치합니다. `@wvb/cli`는 빌드 시점에 번들을 패킹합니다.

<Tabs items={['npm', 'pnpm', 'yarn']}>
  <Tab value="npm">

```sh
npm install @wvb/electron
npm install -D @wvb/cli
```

  </Tab>
  <Tab value="pnpm">

```

  </Tab>

```

  </Tab>

## Register the protocol in the main process

윈도우를 로드하기 전에 `wvb(...)`(`webviewBundle(...)`의 별칭)를 호출하세요. 스킴을 privileged로 등록하고, 소스를 빌드하며, 프로토콜 핸들러와 IPC를 연결합니다.

```ts title="src/main.ts"
import path from 'node:path';
import { app, BrowserWindow } from 'electron';
import { bundleProtocol, localProtocol, wvb } from '@wvb/electron';

const instance = wvb({
  source: {
    builtinDir: path.join(process.resourcesPath, 'bundles'),
  },
  protocols: [
    // 개발: `app-local://app.wvb/...`를 Vite 개발 서버로 프록시하여 핫 리로드를 지원합니다.
    localProtocol('app-local', {
      hosts: { 'app.wvb': MAIN_WINDOW_VITE_DEV_SERVER_URL },
    }),
    // 프로덕션: `app://<bundle>.wvb/...`를 번들에서 직접 제공합니다.
    bundleProtocol('app', { onError: e => console.error('[wvb]', e) }),
  ],
});

async function createWindow() {
  await instance.whenProtocolRegistered();
  const win = new BrowserWindow({
    width: 800,
    height: 600,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      contextIsolation: true,
      nodeIntegration: false,
    },
  });
  await win.loadURL('app://app.wvb');
}

app.whenReady().then(createWindow);
```

URL 형태는 `<scheme>://<bundle-name>.wvb/<path>`입니다. `app://app.wvb/index.html`은 번들 `app`의 파일 `/index.html`로 해석됩니다.

- **`bundleProtocol(scheme, options?)`**는 소스에 있는 번들에서 파일을 직접 제공합니다.
- **`localProtocol(scheme, { hosts })`**는 일치하는 호스트를 개발 서버로 프록시합니다. `hosts`는 필수이며, 호스트를 URL로 매핑하는 `Record<string, string>`(또는 이를 반환하는 함수)입니다.

`app.isPackaged`에 따라 로드할 URL을 선택하세요.

```ts
await win.loadURL(app.isPackaged ? 'app://app.wvb' : 'app-local://app.wvb');
```

`whenProtocolRegistered()`는 모든 프로토콜이 등록되고 `app.whenReady()`가 발생한 후에 resolve됩니다. 화면을 이동하기 전에 항상 await하세요.

`privileges` 옵션으로 프로토콜별 privileges를 재정의하세요.

```ts
bundleProtocol('app', {
  privileges: { stream: true }, // merged over the defaults below
});
```

<Callout type="info">
  각 스킴은 다음 기본값으로 privileged로 등록됩니다. `standard`, `secure`, `bypassCSP`,
  `allowServiceWorkers`, `supportFetchAPI`, `corsEnabled`, `codeCache`는 모두 `true`이고, `stream`은
  `false`입니다. 전체 모델은 [프로토콜 처리](/docs/guide/protocol-handling)를 참고하세요.
</Callout>

## Add the preload script

프리로드는 브리지가 소스, 원격, 업데이터 호출에 사용하는 안전한 전송 계층(`window.wvbElectron.invoke`)을 노출합니다.

```ts title="src/preload.ts"
import { preload } from '@wvb/electron/preload';

preload();
```

`webPreferences.preload`가 컴파일된 프리로드를 가리키게 하고, `contextIsolation: true`와 `nodeIntegration: false`를 유지하세요.

## Call the API from the renderer

UI에서 업데이트를 제어하려면 렌더러 코드에서 브리지를 임포트하세요. 예를 들어 "업데이트 확인" 버튼을 만들 수 있습니다. 브리지는 IPC를 통해 메인 프로세스로 전달되므로 프리로드가 로드되었을 때만 동작합니다.

```ts title="src/renderer.ts"
import { source, remote, updater } from '@wvb/bridge';

// 지금 로컬에 설치된 버전은 무엇인가?
const current = await source.loadVersion('app');

// 원격에 더 새로운 버전이 배포되었는가?
const update = await updater.getUpdate('app');
if (update) {
  const downloaded = await updater.download('app'); // 다운로드 + 검증 + 스테이징
  await updater.install('app', downloaded.version); // 활성화
  // 새 번들을 반영하려면 윈도우를 리로드하세요 — 앱이 직접 처리해야 합니다.
}
```

<Callout type="info">
  `source`, `remote`, `updater`는 `@wvb/bridge`에서 가져오며, `@wvb/bridge`는
  `@wvb/electron/preload`가 설치한 Electron 전송 계층을 자동으로 감지합니다.
  `getUpdate`/`download`의 결과 형태는 [Node API 레퍼런스](/docs/references/node)에 있습니다.
</Callout>

`wvb(...)`가 반환하는 인스턴스에서 메인 프로세스의 동일한 인터페이스에 접근할 수 있습니다.

```ts
instance.source; // BundleSource
instance.remote; // Remote | null   (null unless `updater` is configured)
instance.updater; // Updater | null  (null unless `updater` is configured)
await instance.whenProtocolRegistered();
```

## Configure over-the-air updates

원격을 가리키는 `updater` 블록을 추가하세요. 이 블록이 있어야 `instance.remote`와 `instance.updater`가 활성화됩니다. 블록을 생략하면 렌더러의 `remote.*` / `updater.*` 호출이 "not initialized" 브리지 오류로 실패합니다.

```ts
wvb({
  source: { builtinDir: path.join(process.resourcesPath, 'bundles') },
  updater: {
    remote: { endpoint: 'https://updates.example.com' },
    channel: 'stable',
    // 무결성 및 서명 검증도 여기에 설정합니다:
    // integrityPolicy, integrityChecker, signatureVerifier
  },
  protocols: [bundleProtocol('app')],
});
```

업데이트를 _누가_ 게시했는지 검증하려면 서명 키를 고정하세요.

```ts
updater: {
  remote: { endpoint: 'https://updates.example.com' },
  signatureVerifier: {
    algorithm: 'ed25519',
    key: '/* raw 32-byte or PEM public key */',
  },
},
```

`SignatureAlgorithm`은 `ecdsaSecp256R1`, `ecdsaSecp384R1`, `ed25519`, `rsaPkcs1V15`, `rsaPss` 중 하나입니다. 무결성과 서명에 대한 자세한 내용은 [원격 번들](/docs/guide/remote-bundles)을, 서버를 세우는 방법(테스트용 로컬 서버 포함)은 [원격 서버 구축](/docs/guide/remote)을 참고하세요.

## Where bundles live

`source`는 `builtinDir`(함께 배포되는 읽기 전용)와 `remoteDir`(다운로드된 업데이트)를 받으며, 둘 다 Electron에 맞는 기본값을 가집니다.

| Option       | Default                                                                          |
| ------------ | -------------------------------------------------------------------------------- |
| `builtinDir` | 패키징된 경우 `process.resourcesPath/bundles`, 그 외에는 `process.cwd()/bundles` |
| `remoteDir`  | `app.getPath('userData')/bundles`                                                |

다운로드된 버전이 builtin 버전보다 우선하므로, `updater.download(...)`와 `updater.install(...)` 후에는 설치된 업데이트가 자동으로 제공됩니다. 해석 방식에 대한 자세한 내용은 [번들 소스](/docs/guide/bundle-sources)를 참고하세요.

## Pack and ship bundles

웹 앱을 빌드한 다음, 산출물을 `builtinDir`로 지정한 디렉터리에 패킹하세요.

```sh
# 먼저 렌더러를 빌드한 뒤(예: `vite build`), 다음을 실행하세요:
npx wvb pack ./dist --outfile bundles/app/app_1.0.0.wvb
```

프로토콜이 런타임에 번들을 제공할 수 있도록 `bundles` 디렉터리를 앱과 함께 배포하세요. 이 파일들을 패키지에 스테이징하는 방법은 패키징 도구에 따라 다릅니다 — [패키징](#packaging)을 참고하세요.

## Packaging

`@wvb/electron`을 사용하는 Electron 앱을 패키징하려면 최종 빌드에 두 가지가 포함되어야 합니다. builtin `bundles` 디렉터리가 앱과 함께 배포되어야 하고, 네이티브 `@wvb/node` 바이너리가 런타임에 로드될 수 있도록 ASAR 아카이브 밖에 있어야 합니다. 전용 통합이 두 가지를 모두 처리하고 패키징 시점에 builtin 번들을 설치하므로, `.wvb` 파일을 직접 스테이징할 필요가 없습니다.

## Troubleshooting

- **빈 윈도우 또는 `ERR_FAILED`** — `wvb(...)`가 실행되고 `whenProtocolRegistered()`가 `loadURL` 전에 resolve되는지, 그리고 URL의 번들 이름이 패킹된 파일과 일치하는지 확인하세요(`app://app.wvb` → 번들 `app`).
- **렌더러가 브리지 API에 접근하지 못함** — 프리로드가 로드되지 않았습니다. `webPreferences.preload`와 그것이 `preload()`를 호출하는지 확인하세요.
- **`remote_not_initialized` / `updater_not_initialized`** — `remote.*` / `updater.*` 호출이 있었지만 `wvb(...)`에 `updater` 블록을 전달하지 않았습니다. [무선(OTA) 업데이트](#configure-over-the-air-updates) 설정을 추가하세요.
- **개발에서는 동작하지만 패키징하면 실패함** — `bundles` 리소스나 네이티브 `@wvb/node` 바이너리가 패키지에 포함되지 않았습니다. [패키징](#packaging)을 참고하세요.
