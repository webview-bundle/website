---

`@wvb/electron-forge`는 Webview Bundle을 패키징 단계에 연결하는 Electron Forge 플러그인입니다. 패킹된 `.wvb` 빌트인을 패키징된 앱의 리소스에 설치하고, 네이티브 `@wvb/node` 애드온이 런타임에 항상 사용 가능하도록 도와줍니다. 덕분에 번들 파일을 직접 배치할 필요가 없습니다. 메인 [Electron 가이드](/docs/guide/platforms/electron)와 함께 사용하세요. 이 가이드는 커스텀 프로토콜로 번들을 제공하고 무선(OTA)으로 업데이트하는 방법을 다룹니다.

## 플러그인이 하는 일

이 플러그인은 Forge의 `PluginBase`를 확장하고 `packageAfterCopy`에 훅을 겁니다. 패키징 시점에 Webview Bundle 설정을 확인하고, 빌트인 `.wvb` 번들을 설치합니다(`remote` 타깃에서 다운로드하거나 `@wvb/cli`로 로컬 워크스페이스에서 패킹). 그런 다음 번들을 `<root>/.wvb/builtin/bundles` 아래에 배치하고, 복사된 앱 디렉터리 옆의 패키징된 앱 리소스로 복사합니다. 이 위치가 바로 `@wvb/electron`이 런타임에 빌트인 번들을 읽어 들이는 곳입니다.

정확한 인터페이스는 패키지 소스를 참고하세요: [`@wvb/electron-forge`](https://github.com/webview-bundle/webview-bundle/tree/main/packages/electron-forge).

## forge.config.ts에 추가하기

다른 플러그인과 함께 Forge 설정에 이 플러그인을 등록하세요. 플러그인 클래스는 `WebviewBundlePlugin`입니다(기본 익스포트로도 제공되며 `WvbPlugin`이라는 별칭도 있습니다).

```ts title="forge.config.ts"
import type { ForgeConfig } from '@electron-forge/shared-types';
import { WebviewBundlePlugin } from '@wvb/electron-forge';

const config: ForgeConfig = {
  plugins: [new WebviewBundlePlugin({ bundlesDir: 'bundles', channel: 'beta' })],
};

export default config;
```

플러그인은 `WebviewBundlePluginConfig`에서 옵션을 읽습니다:

| 옵션                      | 타입                | 기본값      | 의미                                                                               |
| ------------------------- | ------------------- | ----------- | ---------------------------------------------------------------------------------- |
| `bundlesDir`              | `string`            | `'bundles'` | 패키징된 앱 리소스 아래의 대상 디렉터리 이름                                       |
| `configFile`              | `string \| boolean` | `true`      | `true`는 자동 검색 후 병합, 경로를 지정하면 명시적으로 로드, `false`는 인라인 전용 |
| `channel`                 | `string`            | —           | 번들을 설치할 릴리스 채널                                                          |
| `throwWhenBuiltinIsEmpty` | `boolean`           | `true`      | 설치된 번들이 하나도 없으면 예외를 던짐                                            |

또한 공유 [Webview Bundle 설정](/docs/config)에서 가져온 인라인 `root`와 `builtin` 설정을 받습니다. `builtin.target`을 지정하지 않으면, 플러그인은 확인된 원격 엔드포인트를 사용하는 `remote` 타깃을 기본값으로 씁니다.

## 네이티브 바이너리와 추가 리소스

플러그인 없이도 두 가지는 반드시 패키징된 앱에 포함되어야 합니다. 빌트인 `bundles` 디렉터리와 네이티브 `@wvb/node` 애드온입니다. 플러그인은 번들을 처리합니다. 네이티브 애드온은 Forge의 `AutoUnpackNativesPlugin`으로 ASAR 아카이브에서 언팩합니다.

```ts title="forge.config.ts"
import { AutoUnpackNativesPlugin } from '@electron-forge/plugin-auto-unpack-natives';
import { WebviewBundlePlugin } from '@wvb/electron-forge';

const config: ForgeConfig = {
  packagerConfig: {
    asar: true,
    extraResource: ['bundles'], // 빌트인 번들을 함께 배포
  },
  plugins: [
    // …vite 플러그인…
    new AutoUnpackNativesPlugin({}), // @wvb/node의 .node 바이너리를 ASAR에서 언팩
    new WebviewBundlePlugin({}),
  ],
};
```

`extraResource: ['bundles']`는 빌트인 번들을 패키징된 앱의 리소스로 배포합니다. `AutoUnpackNativesPlugin`은 `@wvb/node` 네이티브 `.node` 바이너리를 ASAR에서 언팩하여 런타임에 로드할 수 있게 합니다. [`wvb pack`](/docs/guide/commands/pack)을 실행하면 그 `bundles` 디렉터리에 들어갈 `.wvb` 파일을 생성합니다.

## node_modules를 유지해 네이티브 모듈을 번들에 포함하기

Forge Vite 플러그인은 패키지에서 `node_modules`를 제외할 수 있는데, 이때 네이티브 `@wvb/node` 모듈이 빠집니다. 런타임에 `@wvb/node` 바이너리를 찾지 못하는 문제가 발생하면, `packagerConfig.ignore`를 재정의해 `node_modules`를 유지하세요.

```ts title="forge.config.ts"
const config: ForgeConfig = {
  packagerConfig: {
    asar: true,
    extraResource: ['bundles'],
    // 네이티브 @wvb/node 애드온이 번들에 포함되도록 node_modules를 유지합니다.
    ignore: [/^\/(?!node_modules|\.vite|package\.json)/],
  },
};
```

<Callout type="info">
  앱이 개발 환경에서는 실행되지만 패키징하면 창이 비어 있다면, `bundles` 리소스나 네이티브
  `@wvb/node` 바이너리가 빠졌을 가능성이 큽니다. 위의 `extraResource`, `AutoUnpackNativesPlugin`,
  `packagerConfig.ignore` 재정의를 확인하세요.
</Callout>
