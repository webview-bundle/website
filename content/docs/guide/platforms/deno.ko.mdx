Deno Desktop은 `Deno.BrowserWindow`로 네이티브 웹뷰를 렌더링하고 `Deno.serve`로 이를 제공합니다. Webview Bundle이 이 서버의 핸들러가 되므로, 창은 네트워크에서 자산을 가져오는 대신 패킹된 `.wvb` 자산을 오프라인으로 로드합니다.

```ts title="main.ts"
import { webviewBundle, bundleProtocol } from '@wvb/deno-desktop';

const wvb = await webviewBundle({
  protocols: [bundleProtocol({ scheme: 'app' })],
});

const server = Deno.serve(wvb.fetch);
const win = new Deno.BrowserWindow({ url: `http://localhost:${server.addr.port}` });
await win.closed;
```

`@wvb/deno-desktop`은 통합 계층입니다. `@wvb/deno`는 [`@wvb/node`](/docs/references/node)의 FFI 짝으로, 공유 Rust 코어를 구동합니다.

## 구성 방식

창은 `Deno.serve`가 제공하는 단일 로컬 origin을 가리킵니다. 따라서 이 통합은 **단일 origin**이며, 정확히 하나의 프로토콜만 허용합니다. 코어가 각 플랫폼에 도달하는 방식은 [플랫폼 통합](/docs/guide/platform-integration)을, 프로토콜 모델은 [프로토콜 처리](/docs/guide/protocol-handling)를 참고하세요.

## @wvb/deno-desktop

`webviewBundle(config)`를 호출하세요. 이 함수는 `wvb`로 별칭 지정되며 `WebviewBundle` 클래스로 뒷받침됩니다. `BundleSource`(그리고 선택적으로 `Remote`와 `Updater`)를 만들고 `Deno.serve` 호환 `fetch` 핸들러를 반환합니다. 원격에서 최신 번들을 가져오려면 `updater`를 추가하세요.

```ts title="main.ts"
import { webviewBundle, bundleProtocol } from '@wvb/deno-desktop';

const wvb = await webviewBundle({
  protocols: [bundleProtocol({ scheme: 'app' })],
  updater: {
    remote: { endpoint: 'https://bundles.example.com' },
  },
});

const server = Deno.serve(wvb.fetch);
const win = new Deno.BrowserWindow({ url: `http://localhost:${server.addr.port}` });
await win.closed;
```

### 바인딩

`registerBindings(win, wvb)`를 호출하면 네이티브 명령어를 웹 앱에 노출합니다. 이 함수는 `wvbInvoke`라는 이름의 `Deno.BrowserWindow` 바인딩 하나를 등록합니다. 페이지에서는 `window.bindings.wvbInvoke`로 접근할 수 있으며, `source.*`, `remote.*`, `updater.*` 그룹의 모든 `@wvb/bridge` 명령어를 디스패치합니다.

```ts title="main.ts"
import { webviewBundle, bundleProtocol, registerBindings } from '@wvb/deno-desktop';

const wvb = await webviewBundle({
  protocols: [bundleProtocol({ scheme: 'app' })],
});

const server = Deno.serve(wvb.fetch);
const win = new Deno.BrowserWindow({ url: `http://localhost:${server.addr.port}` });

registerBindings(win, wvb); // window.bindings.wvbInvoke를 연결합니다

await win.closed;
```

바인딩은 FFI 경계를 넘어 예외를 던지지 않습니다. 각 호출은 `InvokeResult` 봉투(envelope)로 resolve됩니다.

```ts
type InvokeResult =
  | { ok: true; value: unknown }
  | { ok: false; error: { code?: string; message: string } };
```

`@wvb/bridge` 클라이언트는 `deno` 플랫폼을 감지하면 이 봉투를 벗겨냅니다. 따라서 웹 코드는 다른 플랫폼과 똑같이 `invoke()`, `source.*`, `remote.*`, `updater.*`를 그대로 호출합니다.

```ts title="app.ts (in the webview)"
import { updater } from '@wvb/bridge';

const update = await updater.getUpdate('my-app');
if (update.isAvailable) {
  await updater.download('my-app');
  await updater.install('my-app', update.version);
}
```

## @wvb/deno

`@wvb/deno`는 [`@wvb/node`](/docs/references/node)의 FFI 짝입니다. `Deno.dlopen`을 통해 사전 빌드된 Rust `cdylib`를 로드하고, 동일한 코어 클래스인 `BundleProtocol`, `LocalProtocol`, `Remote`, `BundleSource`, `Updater`를 노출합니다.

각 클래스는 `Disposable`입니다. `free()`로 명시적으로 해제하거나, `using` 선언으로 스코프를 벗어날 때 `[Symbol.dispose]`를 호출하게 하세요.

```ts title="dispose.ts"
import { BundleProtocol } from '@wvb/deno';

// `lib`과 `source`는 아래의 로드 + BundleSource 단계에서 가져옵니다.
// 명시적 정리.
const protocol = new BundleProtocol(lib, source);
const res = await protocol.handle('get', 'app://my-app/index.html');
protocol.free();

// 또는 `using`으로 스코프에 묶인 정리.
{
  using scoped = new BundleProtocol(lib, source);
  await scoped.handle('get', 'app://my-app/index.html');
} // 여기서 [Symbol.dispose]()가 실행됩니다
```

### 네이티브 라이브러리 로드

명시적 경로에서 `cdylib`를 로드하거나, [`@denosaurs/plug`](https://jsr.io/@denosaurs/plug)를 통해 SHA-256으로 검증된 사전 빌드본을 다운로드하세요.

```ts title="load.ts"
import { loadLib, loadLibViaPlug } from '@wvb/deno';

// 1. 이미 디스크에 있는 라이브러리를 로드합니다.
const lib = loadLib('./vendor/wvb/libwvb_deno.dylib');

// 2. sha256으로 검증된 사전 빌드본을 런타임에 다운로드합니다.
const libViaPlug = await loadLibViaPlug();
```

경로를 하드코딩하는 대신 `WVB_DENO_LIB` 환경변수로 `loadLib`가 가리킬 파일을 지정하세요.

```sh
WVB_DENO_LIB=./vendor/wvb/libwvb_deno.dylib deno run -A main.ts
```

또는 설치 서브커맨드로 라이브러리를 미리 벤더링하세요.

```sh
deno run -A jsr:@wvb/deno/install --out vendor/wvb
```

`@wvb/deno/install`은 GitHub Releases에서 cdylib를 다운로드하고 기본적으로 SHA-256 체크섬을 검증합니다. 지원 대상은 다음과 같습니다.

| Target triple               |
| --------------------------- |
| `aarch64-apple-darwin`      |
| `x86_64-apple-darwin`       |
| `aarch64-unknown-linux-gnu` |
| `x86_64-unknown-linux-gnu`  |
| `x86_64-pc-windows-msvc`    |

`--target`으로 특정 대상을 벤더링하세요.

```sh
deno run -A jsr:@wvb/deno/install --out vendor/wvb --target x86_64-unknown-linux-gnu
```

전체 클래스 및 메서드 레퍼런스는 [Deno API 레퍼런스](/docs/references/deno)를 참고하세요.

## 제한 사항

현재 Deno 바인딩에는 두 가지 제약이 있습니다.

**커스텀 verifier 콜백은 지원하지 않습니다.** `Updater`는 **선언적** `signatureVerifier`만 받습니다. 이는 `algorithm`과 `key`를 가진 `SignatureVerifierOptions`입니다.

```ts title="updater.ts"
const wvb = await webviewBundle({
  protocols: [bundleProtocol({ scheme: 'app' })],
  updater: {
    remote: { endpoint: 'https://bundles.example.com' },
    integrityPolicy: 'strict',
    signatureVerifier: {
      algorithm: 'ed25519',
      key: { format: 'raw', data: publicKeyBytes },
    },
  },
});
```

`@wvb/node`에서 사용할 수 있는 커스텀 `integrityChecker` / `signatureVerifier` 함수 콜백은 FFI 경계를 넘을 수 없습니다.

**`HttpOptions.defaultHeaders`는 지원하지 않습니다.** Deno `Remote`에서는 이 옵션을 쓸 수 없습니다. 다른 `HttpOptions` 필드는 여전히 적용됩니다.

```ts title="http.ts"
const wvb = await webviewBundle({
  protocols: [bundleProtocol({ scheme: 'app' })],
  updater: {
    remote: {
      endpoint: 'https://bundles.example.com',
      http: { userAgent: 'my-app/1.0', timeout: 120_000 },
    },
  },
});
```

무결성과 서명이 플랫폼 전반에서 동작하는 방식은 [원격 번들](/docs/guide/remote-bundles)을 참고하세요.
