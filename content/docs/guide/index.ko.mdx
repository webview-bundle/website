title: 소개
description: 웹뷰 기반 프레임워크와 플랫폼을 위한 오프라인 우선 웹 애플리케이션 배포 시스템입니다.
---

Webview Bundle(`wvb`)은 빌드한 웹 자산(HTML/JS/CSS/미디어)을 압축되고 무결성이 검증된 단일
아카이브(`.wvb`)로 묶습니다. 앱은 이 아카이브를 함께 배포하고, 커스텀 URL 스킴을 통해 웹뷰에
제공합니다. 따라서 첫 화면은 네트워크를 기다리지 않습니다. 원격 서버를 설정하면 앱이 더 새로운
번들을 무선(OTA, over-the-air)으로 다운로드하여, 네이티브 앱스토어 배포 없이 갱신된 코드를
전달합니다. 하나의 `.wvb` 포맷이 공유 Rust 코어를 통해 모든 웹뷰 플랫폼에서 동작합니다.

![빌드 산출물을 .wvb 아카이브로 묶어 웹뷰에 제공하고, 원격 서버에서 무선으로 갱신합니다.](https://static.wvb.dev/diagrams/pack-serve-update.svg)

## 빠른 시작

CLI를 설치하고, 빌드 산출물을 묶은 뒤, 로컬에서 미리 봅니다.

<Tabs items={['npm', 'pnpm', 'yarn']}>
  <Tab value="npm">

```sh
npm install --save-dev @wvb/cli
```

  </Tab>
  <Tab value="pnpm">

```sh
pnpm add -D @wvb/cli
```

  </Tab>
  <Tab value="yarn">

```sh
yarn add -D @wvb/cli
```

  </Tab>
</Tabs>

```sh
# ./dist를 .wvb/<pkgName>으로 묶습니다 (--outfile,-O로 변경).
npx wvb pack

# http://localhost:4312에서 로컬 미리 보기용으로 번들을 제공합니다.
npx wvb serve
```

`wvb.config.ts`에서 `defineConfig`로 빌드를 설정하세요.

```ts title="wvb.config.ts"
import { defineConfig } from '@wvb/config';

export default defineConfig({
  pack: {
    // `.wvb`가 자동으로 붙습니다 -> .wvb/app.wvb
    outFile: '.wvb/app',
  },
  serve: {
    port: 4312,
  },
});
```

모든 명령어와 필드는 [CLI 레퍼런스](/docs/guide/cli)와 [`wvb.config` 레퍼런스](/docs/config)를 참고하세요.

## Webview Bundle을 쓰는 이유

- **오프라인 우선.** 리소스가 로컬에 번들로 포함되므로 첫 화면은 네트워크를 기다리지 않습니다.
- **무선 업데이트.** 새 번들 버전을 배포하면 수정 사항이나 기능을 전달합니다. 네이티브 배포가
  필요 없습니다.
- **무결성과 진위성.** 모든 번들은 내부 체크섬을 담고 있습니다. 다운로드는 SHA-2 무결성
  해시(`sha256`, `sha384`, `sha512`)와 디지털 서명(ECDSA, Ed25519, RSA)으로 검증할 수 있습니다.
- **하나의 포맷, 모든 플랫폼.** 동일한 `.wvb` 아카이브가 공유 Rust 코어를 통해 Electron, Tauri,
  Android, iOS, Deno Desktop에서 동작합니다.

설정에서 원격의 무결성 및 서명 검증을 활성화하세요.

```ts title="wvb.config.ts"
import { defineConfig } from '@wvb/config';

export default defineConfig({
  remote: {
    integrity: { algorithm: 'sha256' },
    signature: {
      algorithm: 'ed25519',
      keyFormat: 'pkcs8',
      // privateKey: ...
    },
  },
});
```

전체 스키마는 [원격, 무결성 및 서명 설정](/docs/config/remote)을 참고하세요.

## 패키지

코어는 Rust 크레이트입니다. 각 플랫폼은 얇은 통합 패키지를 통해 코어를 사용합니다.

| Package                      | Version | What it is                                                                                |
| ---------------------------- | ------- | ----------------------------------------------------------------------------------------- |
| [`wvb`](https://docs.rs/wvb) | 0.2.0   | Rust 코어: 번들 포맷, 소스, 원격, 업데이터, 프로토콜, 무결성, 서명                        |
| `@wvb/cli`                   | 0.1.0   | 커맨드라인 도구(바이너리 `wvb`, `webview-bundle`): pack, serve, upload, deploy, 로컬 원격 |
| `@wvb/config`                | 0.1.0   | `wvb.config.ts`를 위한 `defineConfig`                                                     |
| `@wvb/node`                  | 0.1.0   | Node.js용 코어 N-API 바인딩                                                               |
| `@wvb/bridge`                | 0.1.0   | 웹 앱이 네이티브 호스트와 통신하기 위한 브리지                                            |
| `@wvb/electron`              | 0.1.0   | Electron 통합(프로토콜, IPC, 업데이터)                                                    |
| `@wvb/electron-builder`      | Stable  | electron-builder 통합                                                                     |
| `@wvb/electron-forge`        | Stable  | Electron Forge 플러그인                                                                   |
| `wvb-tauri`                  | 0.1.0   | Tauri 통합, crates.io에 Rust 크레이트로 배포됨                                            |
| `@wvb/remote-aws`            | 0.1.0   | AWS용 원격 설정 및 프로바이더                                                             |
| `@wvb/remote-cloudflare`     | 0.1.0   | Cloudflare용 원격 설정 및 프로바이더                                                      |
| `@wvb/remote-local`          | 0.0.0   | 로컬 시뮬레이션을 위한 원격 설정                                                          |

CLI와 함께 플랫폼 통합을 설치하세요.

```sh
# Electron
npm install @wvb/electron

# Tauri (Rust 크레이트)
cargo add wvb-tauri
```

<Callout type="info">
  Tauri 통합은 crates.io의 **`wvb-tauri` 크레이트**입니다. `@wvb/tauri` npm 패키지는 없습니다. 전체
  상태는 [플랫폼 지원](/docs/guide/platform-support)을 참고하세요.
</Callout>
