---
title: 번들 소스
---

**소스**는 번들이 저장되는 기기 내의 위치입니다. 두 가지 유형의 소스가 있습니다. 하나는 앱 패키지 안에 포함되어 배포되는 읽기 전용 빌트인(`builtin`) 소스이고, 다른 하나는 리모트로부터 원격으로 업데이트하는 리모트(`remote`) 소스입니다.

## 빌트인 vs 리모트

웹뷰 번들은 빌트인(`builtin`)과 리모트(`remote`) 번들 소스를 나눕니다.

- **`builtin`** — 앱 패키지 _안에_ 포함되어 배포되는 번들입니다. 읽기 전용이며, 앱을 처음 실행할 때 아직 아무것도 다운로드되지 않은 상태에서 폴백으로 사용됩니다.
- **`remote`** — 리모트 서버에서 다운로드한 번들입니다. 쓰기 가능하며, 앱스토어 릴리스 없이 원격으로 업데이트됩니다.

번들이 두 소스에 모두 존재하면 **리모트가 우선합니다**. 소스는 리모트 매니페스트의 현재 버전을 먼저 확인하여 버전을 결정하며, 리모트에 없으면 빌트인으로 폴백합니다.

---

이러한 분리 덕분에 업데이트가 안전해집니다. 폴백할 수 있는 배포된 빌트인 버전이 항상 존재하며, 다운로드된 리모트 버전은 검증과 설치를 마치면 자연스럽게 빌트인 버전을 대체합니다. 다운로드 및 검증 흐름은 [Over-The-Air](/docs/guide/core-concepts/over-the-air)를 참고하세요.

## 디스크 레이아웃

소스 디렉터리 안에서 각 번들은 자체 폴더를 가지며, 매니페스트는 루트에 위치합니다.

```text
{source_dir}/
├── {name}/
│   └── {name}_{version}.wvb
└── manifest.json
```

예를들어, `app` 번들이 `1.0.0`, `1.1.0` 두 버전이 설치되어 있을 때 디스크 레이아웃은 다음과 같습니다.

```text
{source_dir}/
├── app/
│   ├── app_1.0.0.wvb
│   └── app_1.1.0.wvb
└── manifest.json
```

파일 경로는 항상 `{source_dir}/{name}/{name}_{version}.wvb`입니다. 빌트인 번들과 리모트 번들은 서로 다른 소스이므로, 각각 별도의 매니페스트로 추적됩니다.

<Callout type="info">
  번들 이름과 버전은 경로 구성 요소가 되므로 ASCII `[A-Za-z0-9._-]`로 제한됩니다.
</Callout>

## 매니페스트

`manifest.json`은 소스 디렉터리의 루트에 위치합니다. 각 번들에 대해 존재하는 버전, 해당 메타데이터, 그리고 어느 버전이 현재 버전인지를 기록합니다.

```json title="manifest.json"
{
  "manifestVersion": 1,
  "entries": {
    "app": {
      "versions": {
        "1.0.0": {},
        "1.1.0": {}
      },
      "currentVersion": "1.0.0"
    }
  }
}
```

각 필드는 다음과 같습니다.

| Field                            | Type              | Description                              |
| -------------------------------- | ----------------- | ---------------------------------------- |
| `manifestVersion`                | integer           | 매니페스트 스키마 버전. 항상 `1`입니다.  |
| `entries`                        | object            | 번들 엔트리의 맵                         |
| `entries.{name}.versions`        | object            | 버전별 메타데이터의 맵                   |
| `entries.{name}.currentVersion`  | string (optional) | 이 번들에 대해 제공되는 활성 버전입니다. |
| `entries.{name}.previousVersion` | string (optional) | 현재 버전 직전에 활성이었던 버전입니다.  |

각 버전의 메타데이터는 모든 필드가 선택 사항인 객체입니다. 일반적으로 메타데이터는 번들을 다운로드할 때 리모트 서버의 응답 헤더에서 채워집니다.

| Field          | Type   | Description                                                   |
| -------------- | ------ | ------------------------------------------------------------- |
| `etag`         | string | 다운로드한 번들에 대한 서버의 `ETag` 값입니다.                |
| `integrity`    | string | 무결성 해시. `<alg>:<base64>` 형식입니다(예: `sha256:n4bQ…`). |
| `signature`    | string | 무결성 문자열에 대한 Base64 서명입니다.                       |
| `lastModified` | string | 서버의 `Last-Modified` 값입니다.                              |

### 현재 버전과 이전 버전

`currentVersion`은 소스가 지금 제공하는 버전입니다. 여러 버전이 스테이징되었지만 아직 활성화된 버전이 없을 때는 이 값이 없을 수 있습니다. 버전을 스테이징하면 `currentVersion`을 바꾸지 않은 채 `versions` 아래에 해당 버전이 기록됩니다.

`previousVersion`은 가장 최근 활성화 직전에 현재 버전이었던 버전입니다. 소스는 두 가지 이유로 이 버전을 보관합니다. 첫째, 정상 동작이 확인된 빌드로 롤백할 수 있게 해줍니다. 둘째, 이미 해당 파일을 연 진행 중인 요청이 새 버전이 현재 버전이 되는 동안에도 읽기를 깔끔하게 마칠 수 있도록 이전 파일을 디스크에 유지합니다. 현재 버전과 이전 버전만 보관되며, 더 오래된 버전은 정리됩니다.
