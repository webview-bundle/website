title: Over-The-Air
---

신규 웹뷰 번들을 원격(Over-The-Air)에서 다운로드할 수 있습니다. 네이티브 앱 배포없이도 업데이트된 앱 코드를 웹뷰 번들 리모트 서버를 통해 배포할 수 있습니다.

## 업데이트 라이프사이클

앱 코드를 빌드하여 웹뷰 번들 포맷으로 패키징한 이후 기기에 다운로드되기까지 다섯 단계를 거칩니다.

1. 빌드 결과물을 `.wvb` 포맷으로 패킹합니다.
2. 패킹된 웹뷰 번들 파일을 리모트 서버로 업로드합니다. 무결성 해시와 서명을 추가할 수 있습니다.
3. 업로드한 버전을 배포하면 클라이언트가 업데이트 정보를 불러올 수 있습니다.
4. 클라이언트에서 새로운 버전을 확인한 뒤, 다운로드 합니다.
5. 다운로드된 데이터를 검증하고, 이를 리모트 소스에 설치합니다.

1~3 단계는 CLI 명령어로 실행할 수 있습니다.

```sh
wvb pack                     # `.wvb` 파일 패키징
wvb upload --version 1.2.0   # 리모트 서버로 1.2.0 버전 업로드
wvb deploy --version 1.2.0   # 1.2.0을 현재 버전으로 배포
```

또는, 이용중인 플랫폼/프레임워크에서 더 편한 방식으로 연동할 수 있습니다. 이는 **네이티브** 섹션에서 가이드를 확인해주세요.

## 클라이언트에서 업데이트

클라이언트에서 리모트 번들을 다운받아 리모트 [번들 소스](/docs/guide/core-concepts/bundle-source)에 설치하는 과정을 제어할 수 있습니다. 사용할 수 있는 API는 다음과 같습니다.

- **updater.getUpdate** : 리모트의 현재 버전 정보를 조회(HEAD 요청)해 설치된 버전과 비교합니다. 다운로드는 하지 않으며, 업데이트 가능 여부와 버전 정보를 반환합니다.
- **updater.download** : 새 버전의 번들을 내려받아 검증한 뒤 디스크에 준비합니다. 이 단계에서는 아직 활성화되지 않습니다.
- **updater.install** : 내려받아 둔 버전을 현재 버전으로 활성화합니다. 설치한 뒤부터 해당 버전이 제공됩니다.

다운로드와 설치 과정이 분리되어 있어서, 앱의 비즈니스 요구사항에 맞춰 원하는 타이밍에 신규 번들을 반영할 수 있습니다.

---

Rust 코어 크레이트에서 API를 활용하는 방법입니다.

```rust
use std::sync::Arc;
use wvb::updater::Updater;

let updater = Updater::new(source, remote, None);

let update = updater.get_update("app").await?;
if update.is_available {
  let info = updater.download("app", None).await?;
  updater.install("app", info.version).await?;
  // 이후 새 버전으로 번들 사용 가능
}
```

각 플랫폼/프레임워크에서 업데이트 API를 사용하는 방법은 가이드 문서에서 확인해주세요.

### 무결성

무결성 해시를 사용하면 클라이언트가 다운로드한 번들이 리모트에서 배포된 번들과 데이터가 일치하는지 확인할 수 있습니다. 지원하는 알고리즘(SHA-2)은 `sha256`(기본값), `sha384`, `sha512`이며, `"<alg>:<base64>"` 형식으로 직렬화됩니다.

```text
sha256:n4bQgYhMfWWaL+qgxVrQFaO/TxsrC4Is0V1sFbDwCgg=
```

클라이언트는 다운로드한 번들 데이터로 해시를 다시 계산해 리모트가 알려 준 무결성 해시와 일치하는지 확인합니다. 검증은 설치 전에 수행하며, 일치하지 않으면 설치가 실패합니다. 검증 정책은 세 가지입니다.

- `strict` : 무결성 해시가 반드시 있어야 하며, 없거나 일치하지 않으면 실패합니다.
- `optional`(기본값) : 해시가 있으면 검증하고, 없으면 건너뜁니다.
- `none` : 무결성을 검증하지 않습니다.

### 서명

서명은 번들이 개인 키 소유자가 배포한 것임을 증명합니다. 무결성 해시는 데이터가 특정 해시와 일치하는지만 보장하지만, 서명은 배포자의 진위까지 확인합니다. 서명은 무결성 문자열에 대해 생성되므로, 서명을 사용하려면 무결성을 함께 활성화해야 합니다.

지원하는 알고리즘은 다음과 같습니다.

- ECDSA (P-256, P-384)
- Ed25519
- RSA (PKCS#1 v1.5, PSS)

서명 검증은 선택 사항입니다. 클라이언트에 공개 키로 서명 검증기를 설정하면, 다운로드한 번들의 서명을 무결성 문자열과 대조해 확인하고, 검증에 실패하면 설치를 중단합니다.

## 리모트

리모트는 클라이언트가 번들과 메타데이터를 내려받는 HTTP 서버입니다. 엔드포인트 URL로 구성하며, 번들 목록 조회, 현재 버전 메타데이터 조회(HEAD), 현재 버전 및 특정 버전 다운로드를 제공합니다. 버전·무결성·서명 값은 응답 헤더로 전달됩니다.

### 제공자

번들 서버를 직접 운영하지 않아도 되도록 기본 제공자를 제공합니다.

- **AWS** — `@wvb/remote-aws`
- **Cloudflare** — `@wvb/remote-cloudflare`

리모트 HTTP 규약을 구현하면 직접 제공자를 만들 수도 있습니다. 자세한 내용은 [리모트](/docs/guide/remote) 문서를 참고하세요.

### 채널

채널은 배포를 구분하는 이름표입니다(예: `stable`, `beta`). 클라이언트는 지정한 채널의 현재 버전을 받으며, 채널을 지정하지 않으면 기본 배포를 받습니다.

Rust에서는 `UpdaterConfig`에 채널을 지정합니다.

```rust
use wvb::updater::{Updater, UpdaterConfig};

let config = UpdaterConfig::new().channel("beta");
let updater = Updater::new(source, remote, Some(config));
```

채널은 리모트 요청에 `?channel=beta` 쿼리 파라미터로 전달됩니다.
