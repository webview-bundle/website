신규 웹뷰 번들을 원격에서 다운로드할 수 있습니다. 네이티브 앱 배포없이도 업데이트된 앱 코드를 웹뷰 번들 리모트 서버를 통해 배포할 수 있습니다.

## 업데이트 라이프사이클

앱 코드를 빌드하여 웹뷰 번들 포맷으로 패키징한 이후 기기에 다운로드되기까지 다섯 단계를 거칩니다.

1. 빌드 결과물을 `.wvb` 포맷으로 패킹합니다.
2. 패킹된 웹뷰 번들 파일을 리모트 서버로 업로드합니다. 무결성 해시와 서명을 추가할 수 있습니다.
3. 업로드한 버전을 배포하면 클라이언트가 업데이트 정보를 불러올 수 있습니다.
4. 클라이언트에서 새로운 버전을 확인한 뒤, 다운로드 합니다.
5. 다운로드된 데이터를 검증하고, 이를 리모트 소스에 설치합니다.

1~3 단계는 CLI 명령어로 실행할 수 있습니다.

```sh
wvb pack                     # `.wvb` 파일 패키징
wvb upload --version 1.2.0   # 리모트 서버로 1.2.0 버전 업로드
wvb deploy --version 1.2.0   # 1.2.0을 현재 버전으로 배포
```

또는, 이용중인 플랫폼/프레임워크에서 더 편한 방식으로 연동할 수 있습니다. 이는 **네이티브** 섹션에서 가이드를 확인해주세요.

## 클라이언트에서 업데이트

클라이언트에서 리모트 번들을 다운받아 리모트 [번들 소스](/docs/guide/core-concepts/bundle-source)에 설치하는 과정을 제어할 수 있습니다. 사용할 수 있는 API는 다음과 같습니다.

- **updater.getUpdate** :
- **updater.download** :
- **updater.install** :

다운로드와 설치 과정이 분리되어 있어서, 앱의 비즈니스 요구사항에 맞춰 원하는 타이밍에 신규 번들을 반영할 수 있습니다.

---

Rust 코어 크레이트에서 API를 활용하는 방법입니다.

```rust
use std::sync::Arc;
use wvb::updater::Updater;

let updater = Updater::new(source, remote, None);

let update = updater.get_update("app").await?;
if update.is_available {
  let info = updater.download("app", None).await?;
  updater.install("app", info.version).await?;
  // 이후 새 버전으로 번들 사용 가능
}
```

각 플랫폼/프레임워크에서 업데이트 API를 사용하는 방법은 가이드 문서에서 확인해주세요.

### 무결성

무결성 해시를 사용하면 클라이언트가 다운로드한 번들이 리모트에서 배포된 번들과 데이터가 일치하는지 확인할 수 있습니다. 지원하는 알고리즘(SHA-2)은 `sha256`(기본값), `sha384`, `sha512`이며, `"<alg>:<base64>"` 형식으로 직렬화됩니다.

```text
sha256:n4bQgYhMfWWaL+qgxVrQFaO/TxsrC4Is0V1sFbDwCgg=
```
