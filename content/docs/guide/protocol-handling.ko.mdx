title: 프로토콜 처리
description: 웹뷰 요청이 bundle 프로토콜과 local 프로토콜을 거쳐 .wvb 번들 안의 파일로 매핑되는 방식.
---

프로토콜 핸들러는 웹뷰 요청을 네트워크의 바이트가 아니라 번들의 바이트로 바꿉니다. Webview Bundle은 두 가지 프로토콜을 제공합니다.

- **bundle 프로토콜** — `.wvb` 소스에서 파일을 그대로 제공합니다.
- **local 프로토콜** — 번들러의 개발 서버로 요청을 전달하는 개발용 프록시로, 핫 리로드를 유지합니다.

Rust 코어는 스킴을 등록하거나 검증하지 않습니다. 코어는 URI의 **호스트**와 **경로**만 읽으며, 스킴 자체는 읽지 않습니다. 스킴을 정하고 OS에 등록하는 일은 플랫폼 통합의 몫입니다.

## bundle 프로토콜

요청은 URI의 두 부분을 기준으로 번들 소스에서 해석됩니다. 하나는 **번들 이름**(첫 `.`까지의 호스트)이고, 다른 하나는 **경로**(퍼센트 디코딩된 값)입니다.

```text
app://app.wvb/index.html        -> bundle "app",   file "/index.html"
app://myapp.wvb/assets/logo.png -> bundle "myapp", file "/assets/logo.png"
```

경로 해석은 정적 파일 서버처럼 `index.html`을 채워 넣습니다.

```text
/          -> /index.html       # 끝에 슬래시가 있는 경우
/about     -> /about/index.html # 마지막 세그먼트에 "."이 없는 경우
/a.js      -> /a.js             # 마지막 세그먼트에 "."이 있으면 그대로 제공
```

라벨이 없는 호스트는 bundle-not-found 오류로 실패합니다. 번들 인덱스에 없는 경로는 **404 Not Found**를 반환합니다.

### 메서드, 헤더, 응답

`GET`과 `HEAD`만 제공하며, 그 외의 메서드는 **405 Method Not Allowed**를 반환합니다. `HEAD`는 본문이 비어 있는 헤더를 반환합니다.

파일을 제공할 때 핸들러는 인덱스에 저장된 해당 파일의 헤더를 재생한 뒤, `Content-Type`(인덱스 엔트리에서 가져옴)과 `Content-Length`(압축 해제된 크기)를 덮어씁니다.

```text
GET app://app.wvb/assets/app.js
-> 200 OK
   cache-control: public, max-age=31536000   # 인덱스에서 재생됨
   content-type: application/javascript        # 인덱스 엔트리에서 가져옴
   content-length: 84213                       # 압축 해제된 크기
```

<Callout type="info">
  헤더는 번들을 빌드할 때 파일별로 저장됩니다. 따라서 빌드 시점에 설정한 캐싱 지시자와 그 밖의 응답
  헤더가 제공되는 응답에까지 그대로 유지됩니다.
</Callout>

### 범위 요청

웹뷰는 미디어 탐색과 대용량 다운로드 재개를 위해 범위 요청에 의존합니다. `Range` 헤더가 있으면 `Accept-Ranges: bytes`와 `Content-Range`를 담아 **206 Partial Content**를 반환합니다.

```text
GET app://app.wvb/media/clip.mp4
Range: bytes=0-1023
-> 206 Partial Content
   accept-ranges: bytes
   content-range: bytes 0-1023/5242880
   content-length: 1024
```

- 반환되는 각 범위는 `1000 * 1024` 바이트(약 1 MB)로 제한되며, 이보다 큰 요청 범위는 잘립니다.
- 여러 범위는 `multipart/byteranges` 응답으로 반환됩니다.
- 충족할 수 없는 범위는 `Content-Range: bytes */<length>`와 함께 **416 Range Not Satisfiable**를 반환합니다.

```text
Range: bytes=99999999-
-> 416 Range Not Satisfiable
   content-range: bytes */5242880
```

범위는 압축 해제된 콘텐츠를 기준으로 계산됩니다. 따라서 오프셋은 빌드가 생성한 파일과 일치합니다.

## local 프로토콜

개발용 프록시입니다. `.wvb` 파일을 읽는 대신, 호스트를 localhost 기준 URL로 매핑하고 요청을 전달합니다. 번들러가 핫 리로드되는 자산을 실시간으로 제공하는 동안, 커스텀 스킴은 프로덕션과 동일하게 유지됩니다.

```text
app://myapp/api/data?foo=bar  ->  http://localhost:3000/api/data?foo=bar
```

호스트-URL 매핑을 제공하세요. Electron에서는 `localProtocol(scheme, { hosts })`를 전달합니다.

```ts title="main.ts"
import { localProtocol, wvb } from '@wvb/electron';

const instance = wvb({
  protocols: [
    localProtocol('app', {
      hosts: { 'myapp.wvb': 'http://localhost:3000' },
    }),
  ],
});
```

Tauri에서는 `Protocol::local`에 호스트 맵을 설정합니다.

```rust title="src-tauri/src/lib.rs"
use wvb_tauri::{Config, Protocol};

Config::new().protocol(
  Protocol::local("app").host("myapp.wvb", "http://localhost:3000"),
);
```

매핑이 없는 호스트로 들어온 요청은 resolve 오류로 실패합니다. 프록시는 응답을 캐시하며, 캐시된 응답을 재생하는 방식으로 업스트림의 `304 Not Modified`를 따릅니다.

<Callout type="info">
  개발 중에는 local 프로토콜을 사용하고, 출시 빌드에는 bundle 프로토콜을 사용하세요. 두 프로토콜은
  같은 스킴을 공유하므로, 웹 앱의 URL은 둘 사이에서 바뀌지 않습니다.
</Callout>

## 플랫폼별 스킴

Electron, Tauri, iOS는 실제 커스텀 스킴을 웹뷰에 등록합니다. 반면 Android는 `WebViewClient.shouldInterceptRequest`를 통해 일반 `https://<bundle>.wvb` 요청을 가로채므로, 등록할 커스텀 스킴이 없습니다. 어느 경우든 번들 이름은 호스트 라벨에서, 경로는 URI 경로에서 가져옵니다. 예제는 `app`(Electron, iOS)과 `bundle`(Tauri)을 등록하지만, 이름은 원하는 대로 정할 수 있습니다.

<Tabs items={['Electron', 'Tauri', 'Android', 'iOS']}>
<Tab value="Electron">

`@wvb/electron`은 스킴을 privileged로 등록하고 핸들러를 연결합니다. `bundleProtocol(scheme)`을 `webviewBundle`에 전달하고, `whenProtocolRegistered()`를 await한 뒤 `<scheme>://<bundle>.wvb`를 로드하세요.

```js title="main.cjs"
const path = require('path');
const { app, BrowserWindow } = require('electron');
const { bundleProtocol, wvb } = require('@wvb/electron');

const instance = wvb({
  source: { builtinDir: path.join(__dirname, 'bundles') },
  protocols: [bundleProtocol('app', { onError: e => console.error('[wvb]', e) })],
});

app.whenReady().then(async () => {
  await instance.whenProtocolRegistered();
  const window = new BrowserWindow();
  await window.loadURL('app://app.wvb/index.html');
});
```

스킴은 다음의 기본 권한으로 등록됩니다(`bundleProtocol('app', { privileges })`로 프로토콜별로 재정의 가능).

```ts
{
  standard: true, secure: true, bypassCSP: true,
  allowServiceWorkers: true, supportFetchAPI: true,
  corsEnabled: true, stream: false, codeCache: true,
}
```

권한과 렌더러 브리지는 [Electron 가이드](/docs/guide/platforms/electron)를 참고하세요.

</Tab>
<Tab value="Tauri">

`wvb-tauri` 크레이트는 설정된 `Protocol`마다 비동기 URI 스킴 프로토콜을 등록합니다. `Protocol::bundle(scheme)`을 플러그인 설정에 전달한 뒤, 윈도우가 `<scheme>://<bundle>.wvb`를 가리키게 하세요.

```rust title="src-tauri/src/lib.rs"
use wvb_tauri::{Config, Protocol, Source};

tauri::Builder::default()
  .plugin(wvb_tauri::init(
    Config::new()
      .source(Source::new())
      .protocol(Protocol::bundle("bundle")),
  ))
  .run(tauri::generate_context!())
  .unwrap();
```

`tauri.conf.json`에서 윈도우가 스킴을 가리키게 하세요.

```json title="src-tauri/tauri.conf.json"
{
  "app": {
    "windows": [{ "url": "bundle://app.wvb" }]
  }
}
```

전체 플러그인 설정은 [Tauri 가이드](/docs/guide/platforms/tauri)를 참고하세요.

</Tab>
<Tab value="Android">

Android는 커스텀 스킴을 등록하지 않습니다. 통합은 `WebViewClient`에서 일반 `https://<bundle>.wvb` 내비게이션을 가로챕니다. 동일한 호스트-경로 매핑이 적용되므로, `https://app.wvb/index.html`은 번들 `app`의 파일 `/index.html`로 해석됩니다.

```kotlin
class BundleWebViewClient : WebViewClient() {
  override fun shouldInterceptRequest(
    view: WebView,
    request: WebResourceRequest,
  ): WebResourceResponse? {
    // 요청을 Webview Bundle 핸들러에 넘깁니다.
    return webviewBundle.handle(request)
  }
}
```

동작하는 설정은 [Android 가이드](/docs/guide/platforms/android)를 참고하세요.

</Tab>
<Tab value="iOS">

iOS는 `WKURLSchemeHandler`를 통해 `WKWebView` 설정에 실제 커스텀 스킴을 등록한 뒤, `<scheme>://<bundle>.wvb`를 로드합니다. 최소 배포 대상은 iOS 16입니다.

```swift
let configuration = WKWebViewConfiguration()
configuration.setURLSchemeHandler(bundleSchemeHandler, forURLScheme: "app")

let webView = WKWebView(frame: .zero, configuration: configuration)
webView.load(URLRequest(url: URL(string: "app://app.wvb/index.html")!))
```

동작하는 설정은 [iOS 가이드](/docs/guide/platforms/ios)를 참고하세요.

</Tab>
</Tabs>
