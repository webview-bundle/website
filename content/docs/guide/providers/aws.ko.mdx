title: AWS 프로바이더
description: S3, CloudFront, Lambda@Edge와 선택적 KMS 서명을 사용해 AWS에서 원격 Webview Bundle을 호스팅하고, 제공하고, 프로비저닝합니다.
---

직접 관리하는 AWS 계정에 번들을 게시하고 글로벌 CDN으로 제공합니다. 번들은 **S3**에 저장되고, **CloudFront**가 이를 캐싱하며, 두 개의 **Lambda@Edge** 함수가 원격 HTTP 계약을 S3 읽기로 변환합니다. 그리고 **KMS**로 각 번들에 선택적으로 서명할 수 있습니다.
이 구성 요소들이 구현하는 계약과 프로바이더를 선택하는 방법은 [원격 구축하기](/docs/guide/remote)를 참고하세요.

## 지원 서비스

| Service     | Role                                                                                                                                 |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| S3          | 각 번들 객체와 현재 버전을 기록하는 `deployment.json` 포인터를 저장합니다.                                                           |
| CloudFront  | S3 앞단의 CDN입니다. 배포기는 배포 후 캐시 무효화를 요청할 수 있습니다.                                                              |
| Lambda@Edge | 두 개의 함수입니다. origin-request는 URL을 번들 키로 재작성하고, origin-response는 S3 객체 메타데이터를 계약 응답 헤더로 변환합니다. |
| KMS         | 선택 사항입니다. 각 번들의 무결성 문자열에 서명하여 클라이언트가 게시자를 검증할 수 있게 합니다.                                     |

## 게시 측 설정

`awsRemote()`는 `{ uploader, deployer, signature? }`를 반환합니다. 이를 설정의 `remote` 블록에 펼쳐 넣으세요.

```ts title="wvb.config.ts"
import { defineConfig } from '@wvb/config';
import { awsRemote } from '@wvb/remote-aws';

export default defineConfig({
  remote: {
    endpoint: 'https://bundles.example.com',
    ...awsRemote({
      bucket: 'my-app-bundles',
      aws: {
        region: 'us-east-1',
        profile: 'my-app-deploy',
      },
    }),
  },
});
```

`remote` 스키마 전체는 [원격, 무결성, 서명 설정](/docs/config/remote)을 참고하세요.

### uploader와 deployer 커스터마이징

`uploader`와 `deployer`는 각자의 옵션을 받습니다. uploader는 멀티파트 업로드로 S3에 스트리밍하고, 계약 값을 S3 객체 메타데이터(`webview-bundle-name`, `webview-bundle-version`, 그리고 존재할 경우 `webview-bundle-integrity`와 `webview-bundle-signature`)로 기록합니다. deployer는 `deployment.json`을 GET한 뒤 PUT하며, 배포 후 CloudFront를 무효화할 수 있습니다.

```ts title="wvb.config.ts"
...awsRemote({
  bucket: 'my-app-bundles',
  aws: { region: 'us-east-1' },
  uploader: {
    // 기본값: bundles/${name}/${name}_${version}.wvb
    key: (name, version) => `bundles/${name}/${version}/${name}_${version}.wvb`,
    cacheControl: 'public, max-age=31536000, immutable',
    contentType: 'application/webview-bundle',
  },
  deployer: {
    invalidation: {
      distributionId: 'E1ABCDEF2GHIJK',
    },
  },
}),
```

번들이 이미 존재하고 `force`를 전달하지 않으면, uploader는 `HeadObject` 사전 확인을 실행하고 `BundleAlreadyUploadedError`를 던집니다.

```ts
import { isBundleAlreadyUploadedError } from '@wvb/remote-aws';

try {
  await uploader.upload({ bundle, bundleName, version });
} catch (err) {
  if (isBundleAlreadyUploadedError(err)) {
    // 덮어쓰려면 --force로 다시 실행하세요.
  }
  throw err;
}
```

### KMS로 서명하기

각 번들의 무결성 문자열에 서명하려면 `signature`에 KMS 서명자 설정을 지정하세요. 서명자는 `MessageType: 'DIGEST'`로 KMS를 호출하고 base64 서명을 반환합니다.

```ts title="wvb.config.ts"
import { awsRemote } from '@wvb/remote-aws';

...awsRemote({
  bucket: 'my-app-bundles',
  aws: { region: 'us-east-1' },
  signature: {
    keyId: 'arn:aws:kms:us-east-1:111122223333:key/abcd-1234',
    algorithm: 'ECDSA_SHA_256',
  },
}),
```

서명자를 단독으로 만들려면 `awsKmsSignatureSigner({ keyId, algorithm })`를 사용하세요. 서명 함수를 반환합니다. 서명을 비활성화하려면 `signature: false`를 전달하세요.

<Callout type="warn">
  기본 uploader 키와 프로바이더가 기대하는 키는 일치하지 않습니다. uploader는 `bundles/{name}/{name}
  _{version}.wvb`에 기록하지만, Lambda@Edge 프로바이더는 요청을 `bundles/
  {name}/{version}/{name}_{version}.wvb`로 재작성합니다. 즉 `{version}` 디렉터리가 하나 더 있습니다.
  엔드투엔드 AWS 구성에서는 위 예시처럼 프로바이더의 레이아웃을 만드는 커스텀 uploader `key`를
  설정하거나, 일치하는 S3 레이아웃을 마련하세요. 기본값에 의존하기 전에 패키지 소스와 대조해
  확인하세요.
</Callout>

## Lambda@Edge에서 제공하기

`@wvb/remote-aws-provider`는 계약을 두 개의 CloudFront 이벤트 핸들러로 분리합니다. 각각을 별도의 Lambda에 연결하세요.

```ts title="origin-request.ts"
import { originRequest } from '@wvb/remote-aws-provider/origin-request';

export const handler = originRequest({
  bucketName: 'my-app-bundles',
  region: 'us-east-1',
  // allowOtherVersions: true, // GET /bundles/{name}/{version} 허용, 그렇지 않으면 403
});
```

```ts title="origin-response.ts"
import { originResponse } from '@wvb/remote-aws-provider/origin-response';

export const handler = originResponse();
```

origin-request 핸들러는 `deployment.json`을 읽고, 버전을 해석하며(채널에서 기본 버전으로 폴백), origin URI를 번들 키로 재작성하고, 요청에 태그를 붙입니다. origin-response 핸들러는 S3의 `x-amz-meta-webview-bundle-*` 메타데이터를 `webview-bundle-*` 응답 헤더로 변환합니다. `allowOtherVersions: true`가 아니면 `GET /bundles/{name}/{version}`은 `403`을 반환합니다.

## Pulumi로 프로비저닝하기

`WebviewBundleRemoteProvider`(토큰 `webview-bundle:aws:RemoteProvider`)는 S3 버킷과 정책, Lambda@Edge용 IAM 역할, `us-east-1`에 두 개의 Lambda@Edge 함수(Lambda@Edge에 필수), 그리고 두 함수가 모두 연결된 CloudFront 배포를 생성합니다.

```ts title="index.ts"
import { WebviewBundleRemoteProvider } from '@wvb/remote-aws-provider-pulumi';

const provider = new WebviewBundleRemoteProvider('webview-bundle', {});

export const bucketName = provider.bucketName;
export const distributionDomainName = provider.cloudfrontDistributionDomainName;
export const distributionId = provider.cloudfrontDistributionId;
export const originRequestArn = provider.lambdaOriginRequestArn;
export const originResponseArn = provider.lambdaOriginResponseArn;
```

이 컴포넌트는 배포 시점에 Lambda 코드를 번들링하고 각 함수에 `bucketName`, `region`, `allowOtherVersions`를 주입합니다. 함수는 기본적으로 `nodejs22.x` 런타임을 사용합니다. 함수별 설정은 `lambdaOriginRequest` / `lambdaOriginResponse`로 재정의하세요.

```ts title="index.ts"
new WebviewBundleRemoteProvider('webview-bundle', {
  lambdaOriginRequest: {
    runtime: 'nodejs22.x',
    architecture: 'arm64',
    memorySize: 256,
  },
});
```

Lambda@Edge는 `us-east-1`을 요구합니다. Pulumi AWS 프로바이더를 해당 리전으로 설정하세요.

```ts title="index.ts"
import * as aws from '@pulumi/aws';

const usEast1 = new aws.Provider('us-east-1', { region: 'us-east-1' });
```
