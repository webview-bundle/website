---
title: 로컬 프로바이더
description: 클라우드 프로바이더를 도입하기 전에, 내 컴퓨터에서 파일시스템 기반 원격 서버를 실행해 업로드, 배포, 무선(OTA) 업데이트로 이어지는 전체 흐름을 테스트하세요.
---

내 컴퓨터에서 실행되는 파일시스템 기반 원격 서버입니다. 클라우드 프로바이더와 동일한 HTTP 규약을 구현하므로, 인프라 없이도 전체 수명 주기(패킹, 업로드, 배포, 무선(OTA) 업데이트)를 실행할 수 있습니다. 이 서버로 개발한 다음, 프로덕션에서는 [클라우드 프로바이더](/docs/guide/remote)로 전환하세요.

<Callout type="warn">
  개발과 테스트 전용입니다. 번들을 한 컴퓨터에 일반 파일로 저장하며, `wvb remote local`은
  `localhost`에 바인딩됩니다. 프로덕션 트래픽에는 [AWS](/docs/guide/providers/aws) 또는
  [Cloudflare](/docs/guide/providers/cloudflare) 프로바이더를 사용하세요.
</Callout>

## 두 개의 패키지

둘 다 `0.0.0` 버전입니다.

| Package                      | Role                                                                                                               |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `@wvb/remote-local`          | 게시(publish) 측 클라이언트입니다. `localRemote()`는 `wvb.config`에 연결하는 `uploader`와 `deployer`를 생성합니다. |
| `@wvb/remote-local-provider` | 서버입니다. 저장된 번들을 HTTP로 제공하는 [Hono](https://hono.dev) 앱입니다. CLI가 대신 실행해 줍니다.             |

이 분리 구조는 모든 백엔드에서 동일합니다. 일반 패키지는 번들을 푸시하고, `-provider` 패키지는 번들을 제공합니다. 공유 규약은 [원격 서버 만들기](/docs/guide/remote)를 참고하세요.

## 게시 측 설정하기

`{ uploader, deployer }`를 반환하는 `localRemote()`를 `remote` 블록에 펼쳐 넣으세요.

```ts title="wvb.config.ts"
import { defineConfig } from '@wvb/config';
import { localRemote } from '@wvb/remote-local';

export default defineConfig({
  remote: {
    endpoint: 'http://localhost:4313',
    bundleName: 'app',
    ...localRemote(),
  },
});
```

`localRemote()`는 선택적 필드 하나를 받습니다.

| Option    | Type     | Default        | Description                                          |
| --------- | -------- | -------------- | ---------------------------------------------------- |
| `baseDir` | `string` | `~/.wvb/local` | 번들과 배포 상태를 디스크에 저장하는 디렉터리입니다. |

`baseDir`을 설정하면 프로젝트의 번들을 기본 저장소와 분리할 수 있습니다.

```ts title="wvb.config.ts"
import os from 'node:os';
import path from 'node:path';
import { defineConfig } from '@wvb/config';
import { localRemote } from '@wvb/remote-local';

export default defineConfig({
  remote: {
    endpoint: 'http://localhost:4313',
    bundleName: 'app',
    ...localRemote({
      baseDir: path.join(os.homedir(), '.wvb', 'my-app'),
    }),
  },
});
```

`remote` 블록 전체(`endpoint`, `bundleName`, `packBeforeUpload`, 무결성, 서명)는 [원격 설정 레퍼런스](/docs/config/remote)를 참고하세요.

## 디스크 레이아웃

모든 것은 `{baseDir}/bundles` 아래에 기록되며, 번들 이름마다 디렉터리 하나가 만들어집니다.

```text
{baseDir}/
└── bundles/
    └── {name}/
        ├── {name}_{version}.wvb       # 번들 바이너리 (업로드된 버전마다 하나)
        ├── {name}_{version}.json      # 버전별 메타데이터: { integrity?, signature? }
        └── deployment.json            # 채널별로 어떤 버전이 현재 버전인지 기록
```

번들 `app`의 `1.1.0`을 기본 저장소에 업로드하고 배포한 뒤의 모습입니다.

```text
~/.wvb/local/
└── bundles/
    └── app/
        ├── app_1.1.0.wvb
        ├── app_1.1.0.json
        └── deployment.json
```

`deployment.json`은 현재 버전을 기록합니다. 채널 없이 배포하면 기본 `version`을 설정하고, 채널과 함께 배포하면 `channels[channel]`을 설정합니다. 따라서 채널마다 서로 다른 버전을 가리킬 수 있습니다.

```json title="deployment.json"
{
  "name": "app",
  "version": "1.1.0",
  "channels": {
    "beta": "1.2.0"
  }
}
```

## 서버 실행하기

`@wvb/remote-local-provider`는 Hono 앱을 빌드하기만 하고 포트에 바인딩하지는 않습니다. `wvb remote local`이 그 앱을 Node HTTP 서버로 제공합니다.

```sh
wvb remote local
```

기본적으로 `~/.wvb/local`을 `http://localhost:4313`에서 제공합니다. 플래그는 다음과 같습니다.

| Flag                     | Default        | Description                                                               |
| ------------------------ | -------------- | ------------------------------------------------------------------------- |
| `--base-dir`             | `~/.wvb/local` | 제공할 디렉터리입니다. `localRemote()`에 설정한 `baseDir`과 일치시키세요. |
| `--port`, `-P`           | `4313`         | 수신 대기할 포트입니다(`PORT`도 읽습니다).                                |
| `--hostname`, `-H`       | `localhost`    | 바인딩할 호스트입니다(`HOSTNAME`도 읽습니다).                             |
| `--allow-other-versions` | `false`        | 배포된 버전 외의 버전 다운로드를 허용합니다.                              |
| `--silent`               | `false`        | 서버 로깅을 억제합니다.                                                   |

설정에 지정한 것과 같은 경로를 전달하세요.

```sh
wvb remote local --base-dir ~/.wvb/my-app --port 4313
```

`remote` 명령어 그룹 전체(`remote list`, 별칭 `remote ls`)는 [CLI 레퍼런스](/docs/guide/cli)를 참고하세요.

### allowOtherVersions

기본적으로 서버는 배포된 버전만 제공합니다. `GET /bundles/{name}/{version}`은 `403 Forbidden`을 반환하는데, 이는 클라우드 프로바이더와 동일하며 클라이언트를 특정 버전에 고정해 둡니다.

```sh
curl -i http://localhost:4313/bundles/app/1.0.0
# HTTP/1.1 403 Forbidden
```

이 옵션을 켜면 버전별 경로가 열립니다. 롤백을 테스트하거나 이전 번들을 살펴볼 때 유용합니다.

```sh
wvb remote local --allow-other-versions
```

```sh
curl -i http://localhost:4313/bundles/app/1.0.0
# HTTP/1.1 200 OK
# Webview-Bundle-Name: app
# Webview-Bundle-Version: 1.0.0
```

프로바이더 앱을 직접 빌드할 때는 `allowOtherVersions: true`를 전달하세요.

```ts title="server.ts"
import { wvbRemote } from '@wvb/remote-local-provider';

const app = wvbRemote({
  baseDir: '~/.wvb/local',
  allowOtherVersions: true,
});
```

<Callout type="info">
  `allowOtherVersions`를 비활성화하면 배포된 버전만 `GET /bundles/{name}`으로 접근할 수 있습니다.
  버전별 경로는 이 옵션을 켜기 전까지 `403` 뒤에 숨겨진 상태로 유지됩니다.
</Callout>

## 전체 흐름 따라가기

로컬 서버를 상대로 OTA 업데이트를 처음부터 끝까지 테스트해 봅니다.

<Callout type="info">
  `wvb upload`는 기본적으로 배포하지 않습니다. `--deploy`를 전달하지 않으면 `false`입니다. `wvb
  upload --deploy`로 업로드와 현재 버전 지정을 한 번에 하거나, `wvb deploy`를 따로 실행하세요.
</Callout>

위에서 보여준 대로 `localRemote()`를 연결한 다음, 패킹하고 업로드하고 배포하세요. `packBeforeUpload`가 기본값이면 `wvb upload`가 대신 패킹하므로, 명시적인 `wvb pack`은 선택 사항입니다.

```sh
wvb pack ./dist
wvb upload --version 1.1.0 --deploy
```

같은 저장소를 상대로 서버를 시작하고, 번들이 목록에 나오는지 확인하세요.

```sh
wvb remote local
```

```sh
curl http://localhost:4313/bundles
# [{"name":"app","version":"1.1.0"}]
```

```sh
curl -I http://localhost:4313/bundles/app
# Webview-Bundle-Name: app
# Webview-Bundle-Version: 1.1.0
# Content-Type: application/webview-bundle
```

앱의 업데이터가 `http://localhost:4313`을 가리키게 하세요. 업데이터는 `GET /bundles/{name}`을 호출하고, 번들을 검증한 뒤, `remote` 소스에 설치합니다. 원격 대상은 플랫폼 통합에서 설정하세요. [Electron](/docs/guide/platforms/electron), [Tauri](/docs/guide/platforms/tauri), [Android](/docs/guide/platforms/android), [iOS](/docs/guide/platforms/ios) 가이드를 참고하세요.

버전을 올리고 다시 배포하면 업데이트가 게시됩니다. 다음 업데이터 확인 시점에 이를 가져갑니다.

```sh
wvb upload --version 1.2.0 --deploy
```

`--channel`을 전달하면 기본값 대신 채널로 배포합니다.

```sh
wvb deploy --version 1.2.0 --channel beta
curl 'http://localhost:4313/bundles?channel=beta'
# [{"name":"app","version":"1.2.0"}]
```
