---
title: wvb extract
description: .wvb 아카이브의 파일을 디스크로 다시 풀어 번들에 담긴 내용을 확인합니다.
---

`wvb extract`는 `.wvb` 아카이브를 읽어 그 안의 파일을 디스크로 다시 씁니다. 번들이 무엇을 담고 있는지 확인하거나, 두 번들을 비교하거나, 번들에 들어간 자산을 복원할 때 사용하세요.

## Usage

추출할 번들 파일을 전달하세요. 이 명령어는 풀어낸 파일을 출력 디렉터리에 쓰며, 기본값은 `.wvb/<basename without .wvb>`입니다.

```sh
wvb extract ./build/app.wvb
wvb extract ./build/app.wvb --outdir ./unpacked
wvb extract ./build/app.wvb --outdir ./unpacked --clean
```

`--no-write`를 전달하면 디스크를 건드리지 않고 추출을 실행합니다. 번들이 문제없이 읽히는지 검증할 때 유용합니다.

```sh
wvb extract ./build/app.wvb --no-write
```

## Options

| Option     | Aliases | Default                        | Description                                                       |
| ---------- | ------- | ------------------------------ | ----------------------------------------------------------------- |
| `FILE`     | —       | required                       | 추출할 번들 파일.                                                 |
| `--outdir` | `-O`    | `.wvb/<basename without .wvb>` | 풀어낸 파일을 저장할 대상 디렉터리.                               |
| `--clean`  | —       | `false`                        | 출력 디렉터리가 이미 있으면 먼저 삭제합니다.                      |
| `--write`  | —       | `true`                         | `--no-write`를 전달하면 파일을 쓰지 않고 추출을 시뮬레이션합니다. |
| `--cwd`    | —       | `process.cwd()`                | 경로를 해석하는 데 사용하는 작업 디렉터리.                        |

<Callout type="info">
  `wvb extract`에는 `--config` 플래그가 없습니다. 전달한 번들 파일과 위의 플래그만 읽으며,
  `wvb.config` 파일은 불러오지 않습니다. 다만 상대 경로를 해석하기 위한 `--cwd`는 받습니다.
</Callout>

전역 플래그인 `--color`, `--log-level`, `--log-verbose`는 다른 모든 명령어에서와 마찬가지로 여기서도 적용됩니다. 불리언 플래그는 `--flag`, `--flag=true|false`, 그리고 `--no-flag` 부정 형태를 받습니다. 따라서 `--no-write`는 기본 동작인 `--write`를 끕니다.
