`wvb` 명령줄 도구는 빌드된 웹 자산을 `.wvb` 번들로 묶습니다. 또한 원격 서버에서 번들을 업로드하고, 배포하고, 다운로드하고, 살펴보는 전체 원격 업데이트 워크플로를 처리합니다. 이 도구는 `@wvb/cli` 패키지로 제공되며, 동일하게 동작하는 두 개의 바이너리 `wvb`와 `webview-bundle`을 노출합니다.

개발 의존성으로 설치하세요.

<Tabs items={['npm', 'pnpm', 'yarn']}>
  <Tab value="npm">

```sh
npm install -D @wvb/cli
npx wvb --help
```

  </Tab>
  <Tab value="pnpm">

```sh
pnpm add -D @wvb/cli
pnpm wvb --help
```

  </Tab>
  <Tab value="yarn">

```sh
yarn add -D @wvb/cli
yarn wvb --help
```

  </Tab>
</Tabs>

대부분의 명령어는 작업 디렉터리에서 찾은 [`wvb.config`](/docs/config) 파일에서 기본값을 읽습니다. 따라서 일반적인 프로젝트에서는 인자 없이 `wvb pack`이나 `wvb upload`를 실행합니다. 각 명령어에는 전체 옵션 레퍼런스를 담은 전용 페이지가 있습니다. JavaScript에서 동일한 로직을 호출하려면 [프로그래밍 방식 API](/docs/guide/cli-programmatic)를 참고하세요.

## 전역 플래그

세 가지 플래그가 모든 명령어에 적용됩니다. 이 플래그는 출력 형식과 로깅만 제어합니다.

| 플래그          | 값 / 타입                                 | 기본값  | 환경변수    | 설명                                                              |
| --------------- | ----------------------------------------- | ------- | ----------- | ----------------------------------------------------------------- |
| `--color`       | `off` \| `on` \| `auto`                   | `auto`  | `COLOR`     | 출력의 색상 모드입니다. `auto`는 TTY나 CI 환경에서 색상을 켭니다. |
| `--log-level`   | `debug` \| `info` \| `warning` \| `error` | `info`  | `LOG_LEVEL` | 출력할 최소 로그 레벨입니다.                                      |
| `--log-verbose` | boolean                                   | `false` | —           | 타임스탬프와 카테고리를 포함한 상세 로깅입니다.                   |

<Callout type="info">
`--config`(`-C`)와 `--cwd`는 전역 플래그가 **아닙니다**. 이 둘은 각 명령어별로 선언되며 대부분의 명령어에 존재합니다. `extract`는 `--cwd`를 받지만 `--config`는 없고, `remote local`은 둘 다 받지 않습니다. 존재하는 경우 `--config <path>`는 특정 설정 파일을 가리키고, `--cwd <dir>`는 경로를 해석할 때 사용하는 디렉터리를 바꿉니다.
</Callout>

불리언 플래그는 `--flag`, `--flag=true|false`, 그리고 부정형인 `--no-flag`를 받습니다. 예를 들어 `--no-write`와 `--no-pack`은 `--write`와 `--pack`의 기본값을 끕니다.

## 설정 파일 탐색

명령어는 작업 디렉터리에 있는 `wvb.config` 파일에서 기본값을 로드합니다. 특정 파일을 가리키려면 `--config <path>`를 전달하세요. 전달하지 않으면 `wvb`는 다음 순서로 첫 번째로 일치하는 파일을 찾습니다.

```text
wvb.config.js
wvb.config.cjs
wvb.config.mjs
wvb.config.ts
wvb.config.cts
wvb.config.mts
webview-bundle.config.js
webview-bundle.config.cjs
webview-bundle.config.mjs
webview-bundle.config.ts
webview-bundle.config.cts
webview-bundle.config.mts
wvb.config.json
wvb.config.jsonc
```

`wvb.config`와 `webview-bundle.config` 기본 이름은 모두 `.js`, `.cjs`, `.mjs`, `.ts`, `.cts`, `.mts` 확장자를 지원합니다. `.json`과 `.jsonc` 확장자는 `wvb.config` 기본 이름에서만 지원합니다. 전체 스키마는 [설정](/docs/config)을 참고하세요.
