---

`updater` 명령어 그룹은 웹뷰 내부에서 실행되는 웹 코드에서 무선(OTA) 업데이트 흐름을 구동합니다. 이 그룹으로 더 새로운 번들이 있는지 확인하고, 스테이징한 뒤, 네이티브 호스트에서 활성화하세요. 각 메서드는 타입이 지정된 `invoke()` 호출이며, 대응하는 네이티브 업데이터 핸들러로 라우팅됩니다. 따라서 `invoke()`를 직접 다룰 일은 거의 없습니다.

이 명령어들 뒤에 있는 OTA 개념은 [원격 번들](/docs/guide/remote-bundles)을 참고하세요. bridge가 호출하는 네이티브 업데이터는 [Node 업데이터 API](/docs/references/node/updater)를 참고하세요. bridge 전체는 [Bridge 개요](/docs/references/bridge)를 참고하세요.

```ts
import { updater } from '@wvb/bridge';
```

## Methods

| Method                         | Purpose                                |
| ------------------------------ | -------------------------------------- |
| `listRemotes()`                | 원격 서버와 그 번들 정보를 나열합니다. |
| `getUpdate(bundleName)`        | 업데이트가 있는지 확인합니다.          |
| `download(bundleName)`         | 설치할 업데이트를 스테이징합니다.      |
| `install(bundleName, version)` | 스테이징된 버전을 활성화합니다.        |

## BundleUpdateInfo

`getUpdate()`는 `BundleUpdateInfo`로 resolve됩니다. 이 값은 원격 번들과, 로컬에 설치된 버전과의 비교 결과를 설명합니다.

```ts
type BundleUpdateInfo = {
  name: string;
  version: string;
  localVersion?: string;
  isAvailable: boolean;
  etag?: string;
  integrity?: string;
  signature?: string;
  lastModified?: string;
};
```

업데이트를 스테이징하기 전에 `isAvailable`을 확인하세요. `version` 필드는 원격 서버가 제공하는 버전이고, `localVersion`은 호스트에 현재 설치된 버전입니다(있는 경우).

## Update flow

일반적인 OTA 확인은 더 새로운 번들을 스테이징하고 설치한 뒤, 웹뷰를 다시 로드하여 해당 번들을 제공합니다.

```ts
import { updater } from '@wvb/bridge';

async function checkForUpdate(bundleName: string) {
  const update = await updater.getUpdate(bundleName);
  if (!update.isAvailable) {
    return;
  }

  await updater.download(bundleName);
  await updater.install(bundleName, update.version);

  // 새로 설치된 번들을 웹뷰가 제공하도록 다시 로드합니다.
  window.location.reload();
}
```

<Callout type="info">
  네이티브 호스트에 업데이터가 설정되어 있지 않으면, 호출은 `code`가
  `BridgeErrorCode.UpdaterNotInitialized`인 `BridgeError`로 reject됩니다. 오류 처리는 [Bridge
  개요](/docs/references/bridge)를 참고하세요.
</Callout>
