---

`@wvb/bridge/testing` 하위 경로를 사용하면 실제 네이티브 호스트 없이 브리지를 호출하는 웹 코드를 실행할 수 있습니다. 코드가 호출하는 명령어에 핸들러를 등록하고, 감지된 플랫폼을 강제로 지정하며, 테스트 사이에 목(mock)을 정리하세요. 목 명령어 키는 `'source.loadVersion'`이나 `'updater.getUpdate'`처럼 점(dot)으로 구분되고 타입이 지정되어 있으므로, 핸들러 시그니처의 타입 검사가 유지됩니다.

이 헬퍼가 대신하는 브리지 표면은 [Bridge 개요](/docs/references/bridge)를 참고하세요.

## Import

런타임 진입점과 별개인 `testing` 하위 경로에서 헬퍼를 가져오세요.

```ts
import { mockInvoke, mockPlatform, mockBridge, clearInvokeMocks } from '@wvb/bridge/testing';
```

## 헬퍼

| 헬퍼               | 시그니처                       | 반환값           |
| ------------------ | ------------------------------ | ---------------- |
| `mockInvoke`       | `mockInvoke(command, handler)` | `Disposable`     |
| `mockPlatform`     | `mockPlatform(type)`           | `Disposable`     |
| `mockBridge`       | `mockBridge(options?)`         | 체이닝 가능한 목 |
| `clearInvokeMocks` | `clearInvokeMocks()`           | `void`           |

### mockInvoke

`mockInvoke(command, handler)`는 단일 명령어에 핸들러를 등록하고 `Disposable`을 반환합니다. `command`는 명령어 그룹에 속한, 점으로 구분되고 타입이 지정된 키입니다. `handler`는 일치하는 호출이 resolve할 값을 만듭니다. 반환값을 dispose하면 해당 목만 제거됩니다.

```ts
import { mockInvoke } from '@wvb/bridge/testing';

const mock = mockInvoke('source.loadVersion', () => '1.2.0');

// 나중에 이 단일 목만 제거합니다.
mock[Symbol.dispose]();
```

### mockPlatform

`mockPlatform(type)`는 `platform.type`을 지정한 값으로 강제하고 `Disposable`을 반환합니다. 웹 코드의 플랫폼별 분기를 구동할 때 사용하세요. 모든 `platform` 속성은 실시간으로 계산되므로, 반환값을 dispose하면 정상적인 감지가 복원됩니다.

```ts
import { mockPlatform } from '@wvb/bridge/testing';

const mock = mockPlatform('ios');
// platform.type === 'ios', platform.isIos === true

mock[Symbol.dispose]();
```

### mockBridge

`mockBridge(options?)`는 플랫폼 목과 명령어 목을 결합한, 체이닝 가능한 목을 반환합니다. 옵션에 `platform`을 넘겨 감지될 호스트를 설정하고, `.mockInvoke(command, handler)`를 체이닝하여 명령어를 등록하며, `.clear()`를 호출하여 목이 등록한 모든 것을 제거하세요.

```ts
import { mockBridge } from '@wvb/bridge/testing';

const bridge = mockBridge({ platform: 'electron' })
  .mockInvoke('updater.getUpdate', () => ({
    name: 'app',
    version: '1.2.0',
    isAvailable: true,
  }))
  .mockInvoke('updater.download', () => undefined);

// 테스트 대상 코드를 실행한 뒤:
bridge.clear();
```

### clearInvokeMocks

`clearInvokeMocks()`는 등록된 모든 명령어 목을 한 번에 제거합니다. 전역 teardown 훅에서 호출하여 어떤 목도 다음 테스트로 새어 나가지 않게 하세요.

## 단위 테스트 예시

이 예시는 [Vitest](https://vitest.dev)를 사용하여 목으로 만든 브리지에 대해 업데이트 확인 함수를 테스트합니다. 목은 코드가 호출하는 `updater.getUpdate`와 `updater.download` 명령어를 제공하고, `.clear()`는 각 테스트 이후 상태를 초기화합니다.

```ts title="update.test.ts"
import { afterEach, expect, test, vi } from 'vitest';
import { updater } from '@wvb/bridge';
import { mockBridge } from '@wvb/bridge/testing';

async function hasUpdate(bundleName: string) {
  const update = await updater.getUpdate(bundleName);
  if (!update.isAvailable) {
    return false;
  }
  await updater.download(bundleName);
  return true;
}

let bridge: ReturnType<typeof mockBridge>;

afterEach(() => {
  bridge.clear();
});

test('downloads when an update is available', async () => {
  const download = vi.fn(() => undefined);
  bridge = mockBridge({ platform: 'electron' })
    .mockInvoke('updater.getUpdate', () => ({
      name: 'app',
      version: '1.2.0',
      isAvailable: true,
    }))
    .mockInvoke('updater.download', download);

  await expect(hasUpdate('app')).resolves.toBe(true);
  expect(download).toHaveBeenCalledOnce();
});

test('skips download when no update is available', async () => {
  const download = vi.fn(() => undefined);
  bridge = mockBridge({ platform: 'electron' })
    .mockInvoke('updater.getUpdate', () => ({
      name: 'app',
      version: '1.1.0',
      isAvailable: false,
    }))
    .mockInvoke('updater.download', download);

  await expect(hasUpdate('app')).resolves.toBe(false);
  expect(download).not.toHaveBeenCalled();
});
```

<Callout type="info">
  테스트 대상 코드가 호출하는 모든 명령어에 핸들러를 등록하세요. 목으로 등록되지 않은 명령어는
  핸들러가 없으므로, 브리지는 resolve하는 대신 `BridgeError`를 노출합니다.
</Callout>
