---
title: wvb serve
description: 단일 .wvb 번들의 파일을 HTTP로 제공하여 패킹한 번들을 브라우저에서 미리 봅니다.
---

`wvb serve`는 로컬 HTTP 서버를 시작해 하나의 `.wvb` 번들을 풀고 그 파일을 제공합니다. 덕분에 패킹한 번들을 브라우저에서 열어 배포 전에 확인할 수 있습니다. 디렉터리 경로는 `index.html`로 해석되며, 이는 웹뷰가 런타임에 번들을 로드하는 방식과 동일합니다.

서버는 기본적으로 `http://localhost:4312`에서 요청을 받습니다. `SIGINT`과 `SIGTERM`을 처리해 정상적으로 종료하므로, `Ctrl+C`로 깔끔하게 중지됩니다.

<Callout type="info">
  `wvb serve`는 단일 번들의 내용을 미리 보여줍니다. HTTP 원격을 상대로 전체 무선(OTA) 업데이트
  루프를 테스트하려면, [`wvb remote local`](/docs/references/cli/remote)로 로컬 업데이트 서버를
  실행하세요. 전체 과정을 처음부터 끝까지 따라가려면 [원격 서버 구축하기](/docs/guide/remote)를
  참고하세요.
</Callout>

## 사용법

```sh
wvb serve                                       # 설정에서 해석한 번들을 제공
wvb serve ./build/app.wvb                        # http://localhost:4312
wvb serve ./build/app.wvb --port 8080 --hostname 0.0.0.0
wvb serve ./build/app.wvb --silent               # 요청 로깅 비활성화
```

`FILE`을 생략하면 `wvb serve`는 [설정](/docs/references/configuration)의 `serve.file`을 사용하고, 그다음에는 해석된 패킹 출력 경로를 사용합니다. `wvb pack`을 실행한 일반적인 프로젝트라면 `wvb serve`만으로 미리 볼 수 있습니다.

## 옵션

| Option       | Aliases | Default         | Description                                                                       |
| ------------ | ------- | --------------- | --------------------------------------------------------------------------------- |
| `FILE`       | —       | 설정에서        | 제공할 번들입니다. `serve.file`을 사용하고, 그다음 해석된 패킹 출력을 사용합니다. |
| `--hostname` | `-H`    | `localhost`     | 바인딩할 호스트명입니다. `HOSTNAME` 환경변수를 읽습니다.                          |
| `--port`     | `-P`    | `4312`          | 요청을 받을 포트입니다. `PORT` 환경변수를 읽습니다. 1에서 65535 사이여야 합니다.  |
| `--silent`   | —       | `false`         | 요청 로깅 미들웨어를 비활성화합니다.                                              |
| `--config`   | `-C`    | —               | 설정 파일 경로입니다.                                                             |
| `--cwd`      | —       | `process.cwd()` | 경로 해석에 사용할 작업 디렉터리입니다.                                           |

불리언 플래그는 `--silent`, `--silent=true|false`, 그리고 부정형 `--no-silent`을 받습니다. 전역 플래그인 `--color`, `--log-level`, `--log-verbose`도 여기에 적용됩니다. [CLI 개요](/docs/references/cli)를 참고하세요.

## 참고

- 서버는 디렉터리 요청을 `index.html`로 해석합니다. 따라서 디렉터리에 매핑되는 클라이언트 측 라우트는 번들의 엔트리 문서를 로드합니다.
- `--hostname 0.0.0.0`은 모든 인터페이스에 바인딩합니다. 네트워크상의 다른 기기에서 번들을 미리 볼 때 유용합니다.
- JavaScript에서 동일한 로직을 호출하려면 [프로그래매틱 API](/docs/references/cli/programmatic)의 `serve` 함수를 사용하세요.
