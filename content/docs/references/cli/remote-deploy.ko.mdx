---

`wvb deploy`는 이미 원격에 올라와 있는 번들 버전을 현재 버전으로 승격합니다. 그러면 번들을 다운로드하는 클라이언트가 그 버전을 받습니다. 번들을 업로드하면 원격에 저장되지만 현재 버전이 되지는 않습니다. 스위치를 켜는 단계가 바로 deploy입니다.

[`wvb upload`](/docs/references/cli/remote-upload)을 실행한 뒤에 사용하세요. 또는 `upload`에 `--deploy`를 넘겨 두 단계를 한 명령어로 합칠 수도 있습니다.

## 사용법

```sh
wvb deploy app --version 1.2.0
wvb deploy app --version 1.2.0 --channel beta
wvb deploy                                     # 설정 / package.json 기본값을 사용합니다
```

번들 이름은 위치 인자 `BUNDLE`이며, 생략하면 설정이나 `package.json`에서 해석한 이름으로 대체됩니다. 버전은 `--version`(`-V`) 플래그로 지정합니다. 버전을 위한 위치 인자는 없습니다.

## 옵션

| Option      | Aliases | Default                       | Description                       |
| ----------- | ------- | ----------------------------- | --------------------------------- |
| `BUNDLE`    | —       | 설정에서 해석                 | 배포할 번들 이름.                 |
| `--version` | `-V`    | 설정 또는 `package.json` 버전 | 현재 버전으로 지정할 버전.        |
| `--channel` | —       | —                             | 배포할 릴리스 채널.               |
| `--config`  | `-C`    | 설정 자동 탐색                | 설정 파일 경로.                   |
| `--cwd`     | —       | `process.cwd()`               | 경로 해석에 사용할 작업 디렉터리. |

전역 플래그 `--color`, `--log-level`, `--log-verbose`도 함께 적용됩니다. 자세한 내용은 [CLI 개요](/docs/references/cli)를 참고하세요.

## 요구 사항

`deploy`를 사용하려면 [설정](/docs/references/configuration/remote)에 `remote.deployer`가 필요합니다. deployer는 원격에서 어떤 버전이 현재 버전인지 기록하는 구성 요소입니다. 따라서 deployer가 없는 원격에는 배포할 수 없습니다.

<Callout type="info">
  버전은 위치 인자가 아니라 `--version`(`-V`) 플래그로 지정합니다. 생략하면 deploy는 설정이나 가장
  가까운 `package.json`에서 해석한 버전을 사용합니다.
</Callout>
