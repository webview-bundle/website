title: wvb download
description: 원격 서버에서 번들을 디스크로 다운로드하거나, 메타데이터를 가져와 출력합니다.
---

`wvb download`는 원격 서버에서 번들을 가져오며, 기본적으로 `.wvb` 파일로 디스크에 저장합니다. 번들 이름과 선택적으로 버전을 전달하세요. 버전을 생략하면 현재 배포된 버전을 다운로드합니다. 이 명령어는 `wvb upload`와 `wvb deploy`가 이끄는 배포 워크플로의 읽기 측면에 해당하며, 원격 서버가 실제로 무엇을 제공하는지 확인할 때 유용합니다.

## 사용법

```sh
wvb download app --endpoint https://updates.example.com
wvb download app 1.2.0 --out ./bundles/app.wvb --overwrite
wvb download app --no-write
```

첫 번째 예제는 현재 배포된 `app` 버전을 작업 디렉터리의 `app.wvb`로 다운로드합니다. 두 번째 예제는 특정 버전(`1.2.0`)을 명시한 경로로 다운로드하며, 기존 파일이 있으면 덮어씁니다. 세 번째 예제는 번들을 가져와 정보를 출력하되 디스크에는 아무것도 쓰지 않습니다.

번들 이름과 엔드포인트는 모두 [`wvb.config`](/docs/references/configuration/remote) 파일 값을 기본값으로 사용하므로, 설정이 갖춰진 프로젝트 안에서는 보통 플래그 없이 `wvb download`만 실행해 현재 버전을 가져올 수 있습니다.

## 옵션

| Option        | Aliases | Default             | Description                                                                     |
| ------------- | ------- | ------------------- | ------------------------------------------------------------------------------- |
| `BUNDLE`      | —       | from config         | 번들 이름(위치 인자). 생략하면 설정에서 확인합니다.                             |
| `VERSION`     | —       | current deployed    | 특정 버전(위치 인자). 생략하면 현재 배포된 버전을 다운로드합니다.               |
| `--out`       | `-O`    | `<bundle name>.wvb` | 출력 파일 경로입니다.                                                           |
| `--endpoint`  | `-E`    | `remote.endpoint`   | 다운로드할 원격 엔드포인트입니다.                                               |
| `--channel`   | —       | —                   | 릴리스 채널입니다.                                                              |
| `--write`     | —       | `true`              | 번들을 디스크에 씁니다. `--no-write`를 전달하면 가져와서 정보만 출력합니다.     |
| `--overwrite` | —       | `false`             | 기존 출력 파일을 덮어씁니다.                                                    |
| `--progress`  | —       | `true`              | 다운로드 진행률 표시줄을 보여줍니다. `--no-progress`를 전달하면 비활성화됩니다. |
| `--config`    | `-C`    | auto-discovery      | 설정 파일 경로입니다.                                                           |
| `--cwd`       | —       | `process.cwd()`     | 경로를 확인할 작업 디렉터리입니다.                                              |

불리언 플래그는 `--flag`, `--flag=true|false`, 그리고 `--no-flag` 부정 형태를 받습니다. 세 가지 전역 플래그(`--color`, `--log-level`, `--log-verbose`)도 여기에 적용됩니다. [CLI 개요](/docs/references/cli)를 참고하세요.

<Callout type="info">
  버전 위치 인자를 지정하면 `wvb download`는 해당 버전을 정확히 요청합니다. 지정하지 않으면 해당
  채널에서 현재 배포된 버전을 다운로드합니다. 현재 버전이 아닌 버전을 제공하려면 원격 서버가 다른
  버전을 허용하도록 설정되어 있어야 합니다.
</Callout>

<Callout type="info">
  `--no-write`는 번들을 가져와 정보를 출력하되 파일을 저장하지 않습니다. 디스크를 건드리지 않고 원격
  서버가 무엇을 제공하는지 확인할 때 사용하세요. 번들 본문을 전혀 다운로드하지 않고 메타데이터만
  보려면 대신 [`wvb remote current`](/docs/references/cli/remote)를 사용하세요.
</Callout>
