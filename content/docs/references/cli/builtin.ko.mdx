---
title: wvb builtin
description: 원격 또는 로컬 대상에서 빌트인 웹뷰 번들을 앱에 설치합니다.
---

`wvb builtin`은 앱이 빌트인 폴백으로 함께 배포할 수 있도록 번들을 출력 디렉터리에 설치합니다. 앱이 오프라인으로 시작하거나 무선(OTA)으로 더 새로운 번들을 가져오기 전에는 이 빌트인 번들을 제공합니다. 이 명령어는 [설정](/docs/references/configuration)의 `builtin.target`에 정의된 소스에서 번들을 가져오며, 기본값은 원격 대상입니다.

이 명령어는 설치된 번들 집합을 설명하는 `manifest.json`과, 번들마다 `<name>/<name>_<version>.wvb` 형태로 배치된 파일을 하나씩 기록합니다. 빌트인 소스와 원격 소스가 함께 동작하는 방식은 [번들 소스](/docs/guide/core-concepts/bundle-source)를 참고하세요.

## 사용법

```sh
# 설정된 대상에서 기본 디렉터리로 설치합니다
wvb builtin

# 원격 엔드포인트에서 지정한 디렉터리로 설치합니다
wvb builtin --endpoint https://updates.example.com --out .wvb/builtin/bundles

# 설치할 번들을 필터링합니다
wvb builtin --include 'app*' --exclude 'internal*'

# 감지된 Android 모듈에 설치합니다
wvb builtin --android

# 드라이 런: 실제로 기록하지 않고 설치될 내용만 보고합니다
wvb builtin --no-write
```

## 옵션

| Option          | Aliases | Default                   | Description                                                                                       |
| --------------- | ------- | ------------------------- | ------------------------------------------------------------------------------------------------- |
| `--out`         | `-O`    | `.wvb/builtin/bundles`    | 출력 디렉터리. 모바일 프리셋을 사용하면 플랫폼 모듈 디렉터리로 변경됩니다.                        |
| `--endpoint`    | `-E`    | `remote.endpoint`         | 번들을 가져올 원격 엔드포인트. 원격 대상에만 적용됩니다.                                          |
| `--channel`     | —       | —                         | 설치할 릴리스 채널. 원격 대상에만 적용됩니다.                                                     |
| `--include`     | —       | —                         | 대상 번들에 적용할 glob 포함 패턴. 반복해서 지정할 수 있습니다.                                   |
| `--exclude`     | —       | —                         | 대상 번들에 적용할 glob 제외 패턴. 반복해서 지정할 수 있습니다.                                   |
| `--clean`       | —       | `true`                    | 설치 전에 출력 디렉터리를 비웁니다. 기존 파일을 유지하려면 `--no-clean`을 전달하세요.             |
| `--concurrency` | —       | CPU 개수, 최대 8개로 제한 | 병렬 다운로드 개수. 원격 대상에만 적용됩니다.                                                     |
| `--android`     | —       | —                         | 모바일 프리셋. 인자 없이 쓰면 앱 모듈을 자동 감지하고, `--android=<dir>`로 명시적으로 지정합니다. |
| `--ios`         | —       | —                         | 모바일 프리셋. 인자 없이 쓰면 프로젝트를 자동 감지하고, `--ios=<dir>`로 명시적으로 지정합니다.    |
| `--write`       | —       | `true`                    | 설치된 파일을 기록합니다. 드라이 런은 `--no-write`를 전달하세요.                                  |
| `--progress`    | —       | `true`                    | 다운로드 진행률 표시줄을 보여줍니다. 원격 대상에만 적용됩니다.                                    |
| `--config`      | `-C`    | (자동 탐색)               | 설정 파일 경로.                                                                                   |
| `--cwd`         | —       | `process.cwd()`           | 경로를 해석할 작업 디렉터리.                                                                      |

불리언 플래그는 `--flag`, `--flag=true|false`, 그리고 `--no-flag` 부정 형태를 받습니다. 따라서 `--no-write`와 `--no-clean`은 각각 `--write`와 `--clean`의 기본값을 끕니다.

## 모바일 프리셋

`--android`와 `--ios`는 일반 디렉터리 대신 플랫폼 프로젝트에 번들을 설치합니다. 각각 인자 없이 전달하면 대상을 자동 감지하고, `--android=<dir>` / `--ios=<dir>`로 전달하면 명시적인 모듈이나 프로젝트 디렉터리를 가리킵니다. `--ios` 프리셋은 `Project.swift`에 `folderReference`를 추가하며, 이는 Tuist 프로젝트에 적용됩니다.

<Callout type="warn">한 번의 실행에서 `--android`와 `--ios`를 동시에 전달할 수 없습니다.</Callout>

## 참고

- 대상 유형은 [설정](/docs/references/configuration)의 `builtin.target`에서 결정되며, `remote`와 `local`의 구별된 유니온입니다. 기본값은 원격 대상입니다.
- `--endpoint`, `--channel`, `--concurrency`, `--progress`는 원격 대상에만 적용됩니다.
- 실행에 성공하면 `manifest.json` 하나와, 설치된 번들마다 `<name>/<name>_<version>.wvb` 파일 하나를 기록합니다.
