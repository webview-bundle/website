---

`Updater`는 번들의 무선(OTA) 업데이트를 처리합니다. [`BundleSource`](/docs/references/node/source)와 [`Remote`](/docs/references/node/remote)를 짝지어 동작합니다. 원격 서버에서 더 새로운 버전을 확인하고, 소스의 원격 디렉터리로 다운로드한 뒤 활성화합니다. 확인, 다운로드, 활성화 단계는 각각 별도의 메서드입니다. 따라서 백그라운드에서 업데이트를 스테이징해 두고 나중에 전환할 수 있습니다. 예를 들어 다음 앱 재시작 시점에 전환하면 됩니다. 전체 흐름은 [원격 번들](/docs/guide/remote-bundles)을 참고하세요.

```ts
import { BundleSource, Remote, Updater } from '@wvb/node';

const updater = new Updater(source, remote, options);
```

## Constructor

`new Updater(source: BundleSource, remote: Remote, options?: UpdaterOptions)`

| Argument  | Type                                           | Description                                                 |
| --------- | ---------------------------------------------- | ----------------------------------------------------------- |
| `source`  | [`BundleSource`](/docs/references/node/source) | 스테이징된 번들과 활성 번들이 디스크에 저장되는 위치입니다. |
| `remote`  | [`Remote`](/docs/references/node/remote)       | 원격 서버용 HTTP 클라이언트입니다.                          |
| `options` | `UpdaterOptions`                               | 채널과 검증 설정입니다(선택).                               |

다운로드와 설치는 번들 단위로 직렬화됩니다. 따라서 같은 번들 이름에 대한 동시 호출은 한 번에 하나씩 실행됩니다.

## Methods

| Method        | Signature                                           | Behavior                                                                                                                   |
| ------------- | --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `listRemotes` | `(): Promise<ListRemoteBundleInfo[]>`               | 원격 서버에서 사용 가능한 번들을 나열합니다(설정된 `channel`을 따릅니다).                                                  |
| `getUpdate`   | `(bundleName): Promise<BundleUpdateInfo>`           | 원격 서버의 현재 버전을 로컬 버전과 비교합니다. 이것이 확인 단계입니다.                                                    |
| `download`    | `(bundleName, version?): Promise<RemoteBundleInfo>` | 버전을 원격 디렉터리에 스테이징합니다. 설정된 경우 무결성과 서명을 검증합니다. 활성화하지는 않습니다.                      |
| `install`     | `(bundleName, version): Promise<void>`              | 스테이징된 버전을 활성화합니다. 다시 검증하고, 현재 버전을 교체하고, 캐시된 디스크립터를 버리고, 오래된 버전을 정리합니다. |

`getUpdate`가 확인 단계입니다. `check`라는 이름의 메서드는 없습니다. `download`와 `install`을 분리해 두면 업데이트 단계를 재시작할 수 있습니다. 네트워크가 가능할 때 바이트를 스테이징한 뒤, 안전한 시점에 활성화하세요.

`download`에서 `version` 인자를 생략하면, 업데이터는 설정된 채널의 현재 원격 버전을 가져옵니다. 특정 버전을 스테이징하려면 그 `version`을 직접 전달하세요.

<Callout type="info">
  `getUpdate`는 `BundleUpdateInfo`를 반환합니다. 원격 서버에 배포된 `version`이 설치된
  `localVersion`과 다르면 `isAvailable`이 `true`입니다. 이 값으로 다운로드와 설치 호출을 제어하세요.
</Callout>

## UpdaterOptions

```ts
type UpdaterOptions = {
  channel?: string;
  integrityPolicy?: IntegrityPolicy;
  integrityChecker?: (data: Uint8Array, integrity: string) => Promise<boolean>;
  signatureVerifier?:
    | SignatureVerifierOptions
    | ((data: Uint8Array, signature: string) => Promise<boolean>);
};

type SignatureVerifierOptions = {
  algorithm: SignatureAlgorithm;
  key: { format: VerifyingKeyFormat; data: string | Uint8Array };
};
```

| Field               | Type                                                                  | Description                                                  |
| ------------------- | --------------------------------------------------------------------- | ------------------------------------------------------------ |
| `channel`           | `string`                                                              | 나열 및 확인 호출 시 원격 서버로 전달되는 릴리스 채널입니다. |
| `integrityPolicy`   | `IntegrityPolicy`                                                     | `'strict'`, `'optional'`(기본값), 또는 `'none'`입니다.       |
| `integrityChecker`  | `(data, integrity) => Promise<boolean>`                               | 내장 SHA-2 비교를 대체하는 커스텀 무결성 검사입니다.         |
| `signatureVerifier` | `SignatureVerifierOptions` or `(data, signature) => Promise<boolean>` | 선언적인 알고리즘·키 검증기 또는 커스텀 콜백입니다.          |

### Integrity policy

`integrityPolicy`는 스테이징된 번들의 무결성 해시를 `download` 중에 어떻게 검사하고 `install` 중에 어떻게 다시 검사할지 결정합니다.

| Policy       | Behavior                                                                             |
| ------------ | ------------------------------------------------------------------------------------ |
| `'strict'`   | 무결성 값이 반드시 있어야 하고 일치해야 합니다. 그렇지 않으면 업데이트가 실패합니다. |
| `'optional'` | 원격 서버가 무결성 값을 제공하는 경우 검증합니다(기본값).                            |
| `'none'`     | 무결성 검사를 완전히 건너뜁니다.                                                     |

무결성은 SHA-2(`sha256`, `sha384`, `sha512`)를 사용하며, `"<alg>:<base64>"` 형식으로 직렬화됩니다. 예를 들면 `sha256:n4bQ…`입니다. 자체 로직을 연결하려면, 내장 검사에 의존하는 대신 `integrityChecker` 콜백을 제공하세요.

### Signature verification

신뢰하는 키로 업데이트가 서명되도록 요구하려면 `signatureVerifier`를 설정하세요. 알고리즘과 공개 키로 선언적으로 제공하거나, 커스텀 콜백을 제공하세요.

```ts
type SignatureAlgorithm =
  | 'ecdsaSecp256R1'
  | 'ecdsaSecp384R1'
  | 'ed25519'
  | 'rsaPkcs1V15'
  | 'rsaPss';

type VerifyingKeyFormat = 'spkiDer' | 'spkiPem' | 'pkcs1Der' | 'pkcs1Pem' | 'sec1' | 'raw';
```

키 형식 제약: `pkcs1Der`와 `pkcs1Pem`은 RSA에만, `sec1`은 ECDSA에만, `raw`는 Ed25519(32바이트 키)에만 적용됩니다.

<Callout type="warn">
  서명은 원본 번들 바이트가 아니라 무결성 문자열의 바이트를 대상으로 합니다. 따라서 서명 검증에는
  무결성 값이 반드시 있어야 합니다. `signatureVerifier`는 무결성 값을 생성하는 무결성 정책과 함께
  사용하세요.
</Callout>

전체 타입 목록은 [열거형, 옵션, 결과 타입](/docs/references/node/types)을, 배포 측 설정은 [Remote, 무결성 및 서명 설정](/docs/config/remote)을 참고하세요.

## Update with integrity and signature checks

엄격한 무결성 정책과 선언적 Ed25519 서명 검증기로 `Updater`를 구성한 뒤, 확인·다운로드·설치 단계를 순서대로 실행하세요.

```ts
import { BundleSource, Remote, Updater } from '@wvb/node';

const source = new BundleSource({
  builtinDir: './bundles/builtin',
  remoteDir: './bundles/remote',
});

const remote = new Remote('https://bundles.example.com');

const updater = new Updater(source, remote, {
  channel: 'stable',
  integrityPolicy: 'strict',
  signatureVerifier: {
    algorithm: 'ed25519',
    key: {
      format: 'spkiPem',
      data: process.env.WVB_PUBLIC_KEY!,
    },
  },
});

const update = await updater.getUpdate('my-app');
if (update.isAvailable) {
  await updater.download('my-app', update.version);
  await updater.install('my-app', update.version);
}
```
