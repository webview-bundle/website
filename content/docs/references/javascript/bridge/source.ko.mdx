`source`는 네이티브 번들 소스에 매핑되는 bridge 명령어 그룹입니다. 네이티브 번들 소스는 호스트가 빌트인 번들과 원격 번들을 저장하는 공간입니다. 각 메서드는 타입이 지정된 `invoke()` 호출이므로, 웹 측에서 타입 검사를 거쳐 대응하는 네이티브 핸들러로 라우팅됩니다. 이 그룹으로 번들을 나열하고, 버전과 메타데이터를 불러오고, 디스크상의 경로를 확인하고, 다운로드한 원격 번들을 관리하세요.

이 명령어들이 호출하는 네이티브 측은 [Node API source 레퍼런스](/docs/references/javascript/node/source)를 참고하세요. `invoke()`, `platform`, `remote` 및 `updater` 그룹, 오류, 테스트 등 나머지 bridge 영역은 [Bridge 개요](/docs/references/javascript/bridge)를 참고하세요.

```ts
import { source } from '@wvb/bridge';

const bundles = await source.listBundles();
const path = await source.resolveFilepath('app');
```

## Methods

| Method                                          | Purpose                                         |
| ----------------------------------------------- | ----------------------------------------------- |
| `listBundles()`                                 | 사용 가능한 번들을 나열합니다.                  |
| `loadVersion(bundleName)`                       | 번들의 활성 버전을 불러옵니다.                  |
| `updateVersion(bundleName, version)`            | 번들의 활성 원격 버전을 설정합니다.             |
| `resolveFilepath(bundleName)`                   | 소스가 제공하는 디스크상의 경로를 확인합니다.   |
| `getBuiltinBundleFilepath(bundleName, version)` | 빌트인 번들 파일의 경로입니다.                  |
| `getRemoteBundleFilepath(bundleName, version)`  | 원격 번들 파일의 경로입니다.                    |
| `loadBuiltinMetadata(bundleName, version)`      | 빌트인 번들의 메타데이터를 불러옵니다.          |
| `loadRemoteMetadata(bundleName, version)`       | 원격 번들의 메타데이터를 불러옵니다.            |
| `unloadDescriptor(bundleName)`                  | 캐시된 번들 디스크립터를 제거합니다.            |
| `removeRemoteBundle(bundleName, version)`       | 다운로드한 원격 번들을 삭제합니다.              |
| `remoteRetainedVersions(bundleName)`            | 보관 중인 원격 버전(현재 및 이전)을 나열합니다. |
| `pruneRemoteBundles(bundleName)`                | 더 이상 보관하지 않는 원격 번들을 정리합니다.   |

<Callout type="info">
bridge의 `ListBundleItem`은 `{ type, item }` 형태로 중첩되어 있어, `@wvb/node`의 평면적인 `ListBundleItem`과 다릅니다. 웹 측에서 작업할 때는 bridge 타입을 사용하세요.
</Callout>

## Managing remote bundles

무선(OTA) 업데이트가 적용되면, 호스트는 현재 원격 버전과 이전 버전들을 함께 보관합니다. 보관 중인 항목을 확인한 다음, 더 이상 필요하지 않은 것을 정리하세요.

```ts
import { source } from '@wvb/bridge';

async function tidyRemoteBundles(bundleName: string) {
  const retained = await source.remoteRetainedVersions(bundleName);
  console.log('retained versions:', retained);

  await source.pruneRemoteBundles(bundleName);
}
```

다운로드한 특정 버전을 삭제하려면 `removeRemoteBundle(bundleName, version)`을 호출하세요.
