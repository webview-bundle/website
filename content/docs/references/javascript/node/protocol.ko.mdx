`@wvb/node`는 웹뷰의 요청을 HTTP 방식의 응답으로 바꾸는 두 가지 프로토콜 핸들러를 제공합니다. `BundleProtocol`은 `BundleSource`에서 자산을 제공하고, `LocalProtocol`은 개발 중에 요청을 localhost 서버로 프록시합니다. 두 핸들러 모두 HTTP 메서드, URI, 선택적 헤더를 받아 `HttpResponse`를 반환합니다. 이 스킴들이 웹뷰 플랫폼에 어떻게 들어맞는지는 [프로토콜 처리](/docs/guide/core-concepts/protocol-handling)를 참고하세요.

두 핸들러는 동일한 호출 형태를 공유합니다.

```ts
handle(
  method: HttpMethod,
  uri: string,
  headers?: Record<string, string>,
): Promise<HttpResponse>
```

`HttpMethod`와 `HttpResponse`는 [타입](/docs/references/javascript/node/types)에 문서화되어 있습니다. `HttpMethod`는 소문자 문자열 유니온(`'get'`, `'head'` 등)입니다. `HttpResponse`는 `{ status, headers, body }`이며, 여기서 `body`는 `Buffer`입니다.

## BundleProtocol

`new BundleProtocol(source: BundleSource)`는 `scheme://bundle_name/path` 요청을 [`BundleSource`](/docs/references/javascript/node/source)에서 제공합니다. 번들 내부에서 요청한 파일을 찾아 그 바이트를 반환합니다.

| Member      | Signature                                                                                      |
| ----------- | ---------------------------------------------------------------------------------------------- |
| Constructor | `new BundleProtocol(source: BundleSource)`                                                     |
| `handle`    | `(method: HttpMethod, uri: string, headers?: Record<string, string>) => Promise<HttpResponse>` |

`BundleProtocol`은 `GET`과 `HEAD` 요청에 응답합니다. `GET`은 파일 내용을 `body`에 담아 반환하고, `HEAD`는 본문 없이 응답 메타데이터만 반환합니다. 요청에 `Range` 헤더가 있으면 핸들러는 상태 `206`으로 부분 응답을 제공합니다.

## LocalProtocol

`new LocalProtocol(hosts: Record<string, string>)`는 `app://host/...` 요청을 localhost URL로 프록시합니다. 각 가상 호스트를 전달할 로컬 오리진에 매핑하세요. 이 방식은 웹뷰가 패킹된 `.wvb` 번들 대신 개발 서버를 가리키는 개발 환경에서 유용합니다.

| Member      | Signature                                                                                      |
| ----------- | ---------------------------------------------------------------------------------------------- |
| Constructor | `new LocalProtocol(hosts: Record<string, string>)`                                             |
| `handle`    | `(method: HttpMethod, uri: string, headers?: Record<string, string>) => Promise<HttpResponse>` |

`LocalProtocol`은 응답을 캐시하며, 업스트림 리소스가 변경되지 않았으면 `304`를 반환합니다.

<Callout type="info">
  프로덕션에서 패킹된 번들을 제공할 때는 `BundleProtocol`을 사용하고, 개발 중에 개발 서버로 프록시할
  때는 `LocalProtocol`을 사용하세요. 두 스킴 중 하나를 플랫폼에 연결하는 방법은 [프로토콜
  처리](/docs/guide/core-concepts/protocol-handling)를 참고하세요.
</Callout>

## 프로토콜로 요청 처리하기

`BundleSource`를 만들고 `BundleProtocol`로 감싼 뒤, 웹뷰의 요청에 응답하세요.

```ts
import { BundleSource, BundleProtocol } from '@wvb/node';

const source = new BundleSource({
  builtinDir: './bundles/builtin',
  remoteDir: './bundles/remote',
});

const protocol = new BundleProtocol(source);

const res = await protocol.handle('get', 'app://my-app/index.html');
console.log(res.status); // 200
console.log(res.headers['content-type']);
res.body; // Buffer
```
