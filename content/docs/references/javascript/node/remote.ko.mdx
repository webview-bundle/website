`Remote`는 [번들 원격](/docs/guide/remote)을 위한 `@wvb/node` HTTP 클라이언트입니다. [Remote HTTP Spec](/docs/guide/remote/http-spec)을 구현한 모든 서버와 통신합니다. 따라서 Node.js에서 사용 가능한 번들을 나열하고, 번들의 메타데이터를 읽고, 번들 바이트를 다운로드할 수 있습니다. [Updater](/docs/references/javascript/node/updater)는 `Remote` 위에서 업데이트를 준비하고 활성화합니다. 원격에 더 낮은 수준으로 접근해야 할 때는 `Remote`를 직접 사용하세요.

## 생성자

```ts
new Remote(endpoint: string, options?: RemoteOptions)
```

원격의 기본 `endpoint`를 전달하세요(예: `https://bundles.example.com`). 클라이언트는 HTTP 규약에 따라 해당 엔드포인트로 요청을 보냅니다.

```ts
import { Remote } from '@wvb/node';

const remote = new Remote('https://bundles.example.com');
```

## 메서드

| Method            | Signature                                                             |
| ----------------- | --------------------------------------------------------------------- |
| `listBundles`     | `(channel?: string): Promise<ListRemoteBundleInfo[]>`                 |
| `getInfo`         | `(bundleName: string, channel?: string): Promise<RemoteBundleInfo>`   |
| `download`        | `(bundleName, channel?): Promise<[RemoteBundleInfo, Bundle, Buffer]>` |
| `downloadVersion` | `(bundleName, version): Promise<[RemoteBundleInfo, Bundle, Buffer]>`  |

`listBundles`는 원격이 제공하는 번들을 나열하며, `channel`로 필터링할 수도 있습니다. `getInfo`는 단일 번들의 현재 메타데이터를 읽습니다. `download`는 번들의 현재 버전을 가져오고, `downloadVersion`은 특정 버전을 가져옵니다. 두 다운로드 메서드 모두 번들 정보, 파싱된 [`Bundle`](/docs/references/javascript/node/bundle), 그리고 원본 `.wvb` 바이트를 담은 `Buffer`로 이루어진 튜플을 반환합니다.

```ts
const [info, bundle, bytes] = await remote.download('my-app', 'stable');

console.log(info.name, info.version);
console.log(bundle.getData('index.html')?.toString('utf8'));
console.log(bytes.byteLength);
```

## RemoteOptions

`RemoteOptions`는 HTTP 전송과 선택적인 다운로드 진행률 콜백을 설정합니다.

```ts
type RemoteOptions = {
  http?: HttpOptions;
  onDownload?: (data: RemoteOnDownloadData) => void;
};

type RemoteOnDownloadData = {
  downloadedBytes: number;
  totalBytes?: number;
  endpoint: string;
};
```

`onDownload`는 바이트가 도착할 때마다 실행됩니다. `totalBytes`는 원격이 콘텐츠 길이를 알려줄 때 설정됩니다.

```ts
const remote = new Remote('https://bundles.example.com', {
  onDownload: ({ downloadedBytes, totalBytes }) => {
    if (totalBytes) {
      console.log(`${Math.round((downloadedBytes / totalBytes) * 100)}%`);
    }
  },
});
```

## HttpOptions

`HttpOptions`는 내부 HTTP 클라이언트를 조정합니다. 모든 필드는 선택 사항입니다.

| Field                | Type                     | Description                                |
| -------------------- | ------------------------ | ------------------------------------------ |
| `defaultHeaders`     | `Record<string, string>` | 모든 요청에 함께 보내는 헤더입니다.        |
| `userAgent`          | `string`                 | `User-Agent` 헤더 값입니다.                |
| `timeout`            | `number`                 | 전체 요청 타임아웃(밀리초 단위)입니다.     |
| `readTimeout`        | `number`                 | 읽기 타임아웃(밀리초 단위)입니다.          |
| `connectTimeout`     | `number`                 | 연결 타임아웃(밀리초 단위)입니다.          |
| `poolIdleTimeout`    | `number`                 | 풀링된 연결의 유휴 타임아웃입니다.         |
| `poolMaxIdlePerHost` | `number`                 | 호스트당 유지하는 최대 유휴 연결 수입니다. |
| `referer`            | `boolean`                | `Referer` 헤더를 보낼지 여부입니다.        |
| `tcpNodelay`         | `boolean`                | 소켓에서 `TCP_NODELAY`를 활성화합니다.     |
| `hickoryDns`         | `boolean`                | Hickory DNS 리졸버를 사용합니다.           |

<Callout type="info">
  기본 요청 타임아웃은 120초입니다. `HttpOptions.timeout`(밀리초)으로 재정의하세요.
</Callout>

```ts
const remote = new Remote('https://bundles.example.com', {
  http: {
    timeout: 30_000,
    userAgent: 'my-app/1.0',
  },
});
```

## 결과 형태

`getInfo`, `download`, `downloadVersion`은 `RemoteBundleInfo`를 반환합니다. `listBundles`는 `ListRemoteBundleInfo` 배열을 반환합니다.

```ts
type ListRemoteBundleInfo = {
  name: string;
  version: string;
};

type RemoteBundleInfo = {
  name: string;
  version: string;
  etag?: string;
  integrity?: string; // "<alg>:<base64>", 예: "sha256:n4bQ…"
  signature?: string; // base64
  lastModified?: string;
};
```

`integrity`와 `signature` 필드는 원격이 `Webview-Bundle-Integrity`와 `Webview-Bundle-Signature` 응답 헤더로 알리는 값을 담습니다. `404`(배포되지 않음)나 `403`(금지됨) 같은 실패 응답을 클라이언트가 어떻게 드러내는지는 [Errors](/docs/guide/remote/http-spec/errors)를 참고하세요.
